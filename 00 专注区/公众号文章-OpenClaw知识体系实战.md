# AI 工具满天飞，我选了最笨的方法：自己搭一个知识体系

上周我刷到一个视频，博主演示他的 AI 助手如何自动写代码、自动发邮件、自动管理日程。弹幕清一色"太强了""求教程"。

我也觉得强。

然后关掉视频，什么都没做。

这种场景在过去几个月反复上演。AI 工具一个接一个冒出来，每个都说自己是"下一个生产力革命"。我把它们收藏、记笔记、加入待办清单——然后清单越来越长，焦虑越来越重，动手越来越少。

直到我想明白一件事：**工具焦虑的根源，不是工具太多，而是我没有自己的坐标系。**

没有坐标系，所有信息都像是"应该学的"，所有工具都像是"不能错过的"。结果就是什么都浅尝辄止，什么都没跑通。

这篇文章记录的，是我用一周时间，从零开始用 OpenClaw 搭建个人知识体系的真实过程。不是教程，不是测评，而是一个中职计算机老师折腾 AI 工具的实战手记。

## 为什么是 OpenClaw？

先说背景。我是一名中职计算机老师，日常教 C 语言和 Word 操作。2024 年开始接触 AI 工具后，走了一条比较典型的弯路：

Cherry Studio → ChatGPT → Claude → 各种 Agent 工具 → 眼花缭乱 → 焦虑

后来在一个技术社区看到有人分享了一条学习路径：

> Cherry Studio + Claude 模型 → Claude Code → OpenClaw

三步，顺序不能倒。先把大模型玩明白，再用 Claude Code 赋能日常工作，最后上 OpenClaw 做自动化。

这条路径打动我的地方在于**它说了"不做什么"**。不是让你同时学十个工具，而是给了一个渐进的逻辑：先理解 AI 的能力边界，再逐步扩展使用场景。

我当时处在第 2 步到第 3 步的过渡期——Claude Code 已经用得比较顺手，开始觉得可以尝试更重的东西了。于是选了 OpenClaw。

---

## 第一步：买服务器、装起来

OpenClaw 需要部署在海外服务器上。这是很多人的第一个门槛——听到"买服务器""SSH 连接"就觉得很难。

说实话，比我想象的简单。

我选的是 RackNerd 的一台 VPS：6 核 CPU、8GB 内存、150GB SSD，年费大约 60 美元。机房选了洛杉矶。下单后邮件收到 IP 地址和 root 密码，就可以开始了。

连接服务器用的是 FinalShell——一个国产的 SSH 工具，全中文界面，左边是终端，右边是文件管理器，对不熟悉命令行的人非常友好。

安装 OpenClaw 按官方文档走，配置好网关模式，一行命令后台运行：

```bash
nohup openclaw gateway run > /root/openclaw.log 2>&1 &
```

然后把 Telegram 联通，手机上就能和它对话了。

整个过程花了大约一个下午。最难的部分不是技术操作，而是**下决心动手**。

---

## 第二步：信息焦虑的「当头一棒」

装好之后，我做了一件现在回想起来很典型的事：去找"最佳实践"。

搜公众号、翻 B 站、看社区分享。结果看到的是什么？

有人展示自己的 OpenClaw 管理着 5 个 AI Agent 协同工作。有人让它自动抓取网页、解析数据、生成报告。还有人一周消耗了十亿 token，花了 1500 美金在上面跑 AI 策略团队。

看得我头皮发麻。

这些功能酷不酷？酷。跟我有关系吗？完全没有。

我只是一个想用 AI 提升教学效率的老师。我不需要管理 5 个 Agent，不需要消耗十亿 token。我需要的是：能搜索信息、能整理笔记、能帮我把学到的东西沉淀下来。

**这就是典型的"找最佳实践变成拖延的挡箭牌"。** 表面上在学习，实际上在回避动手。看别人的炫技只会让我觉得自己的需求太简单、太不值一提，反而更不想开始。

想明白这一点之后，我做了一个决定：关掉所有参考资料，从自己的实际需求出发，先跑起来再说。

## 第三步：搭知识体系——用自己的方式

我给自己的 OpenClaw 知识梳理了五个模块：

**一、基础设施**——服务器怎么买、怎么连、怎么装、怎么后台运行。这一块已经搞定了。

**二、配置与定制**——网络能力（搜索、网页抓取）、文件同步、人格配置。我把网络能力分成了四个层级：

| 层级 | 能力 | 我的状态 |
|------|------|----------|
| L0 | 搜索 + 网页抓取 | 待配 |
| L1 | 无头浏览器 | 待配 |
| L2 | 有头浏览器 + DOM 操作 | 待学 |
| L3 | 截图 + 视觉识别 | 待学 |

大部分还没配。但重要的是，我知道了全貌，不会因为"还有好多不会"而焦虑——因为 L0 就能覆盖 80% 的日常需求，其他的以后再说。

**三、日常使用**——核心命令、多群组管理、Skill 编写。

**四、高级实践**——多 Agent 协作、心跳机制、可重入性。这些目前离我很远，冷藏起来，不看不想。

**五、安全与接入**——API Key 管理、服务器安全。刚需但不紧急。

整个知识体系用一个 Markdown 文件记录，每个知识点标注状态：✅ 已掌握、⬜ 待学、❌ 不推荐。

这么做的好处是**随时知道自己在哪里**。不是"我还有好多要学"的焦虑感，而是"我已经掌握了这些，下一步做那个"的确定感。

---

## 最大的收获：不是技术，是思维方式

搭完这个知识体系，我最大的感触不是学到了什么技术，而是验证了一种学习方式——**先动手，再整理，最后才看别人怎么做。**

传统的学习路径是：看教程 → 学知识 → 动手做。

我这次的路径是：先部署跑通 → 用起来碰壁 → 带着问题去看别人的实践 → 补全自己的知识体系。

区别在哪？**带着问题看信息和漫无目的看信息，吸收效率完全不同。**

比如那篇"十亿 token"的分享，如果我一开始就看，只会觉得恐惧和自卑。但在自己部署完、碰过壁之后再看，我能精准地从中提取出对我有用的三个点：

1. 浏览器能力分层（L0-L3）的思路，帮我理清了能力边界
2. rclone 文件同步方案，比我原来想用的坚果云更适合
3. SOUL.md 人格配置的思路，和我自己的设计理念一致

其他的？跟我无关，跳过。

这就是"六经注我"——信息要跟自己长在一起才有用。没有自己的坐标系，看再多信息也是别人的。

---

## 写在最后

回到开头的问题：AI 工具满天飞，该怎么选？

我的答案是：**别选，先做。**

不用找到最好的工具才开始。不用看完所有教程才动手。不用等别人告诉你最佳实践。

找一个跟你当前需求最近的工具，装起来，用起来，碰壁了再查，查到了再整理。

一周时间，你会拥有一份属于自己的知识体系——不是收藏夹里吃灰的链接，而是从实践中长出来的、带着你自己判断标记的活地图。

这比收藏一百个"AI 工具合集"有用得多。
