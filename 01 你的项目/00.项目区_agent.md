# 项目区代理

## 区域概述

**你的项目**存放所有正在进行或已完成的项目资料——产品设计、客户交付、个人 side project、合作项目等。每个项目一个文件夹，自成体系。

这个区域的核心逻辑是**项目隔离**：不同项目之间的信息默认不互通，除非你主动要求交叉引用。

---

## 区域结构

```
01 你的项目/
├── 00.项目区_agent.md        ← 本文件（区域规则）
├── [项目A]/                  ← 每个项目独立文件夹
│   ├── README.md             ← 项目概览（推荐）
│   ├── [设计文档]
│   ├── [会议纪要]
│   └── [交付物]
├── [项目B]/
└── _已归档/                  ← 已结项的项目
```

---

## 区域规则

### 项目隔离

| 规则 | 说明 |
|------|------|
| 默认隔离 | AI 不主动跨项目引用信息 |
| 显式授权 | 用户说「参考项目 A 的做法」时才可跨项目读取 |
| 交付物保护 | 涉及客户或合作方的交付物，修改前必须确认 |

这条规则的存在是因为：项目之间可能涉及不同客户、不同保密等级。宁可多问一句，不要想当然地混用。

### 项目生命周期

```
创建 → 活跃 → 收尾 → 归档
```

- **创建**：新建项目文件夹时，建议同时写一个 README.md 说明项目背景和目标
- **活跃**：文件正常读写，AI 在此项目范围内自由操作
- **收尾**：项目完成后，整理产出清单，确认无遗留
- **归档**：移入 `_已归档/`，之后只读不写

### 操作规范

| 操作 | 权限 |
|------|------|
| 读取项目内文件 | 自由，无需额外权限 |
| 在项目内新建文件 | 自由 |
| 修改项目文档 | 说明变更内容 |
| 删除文件 | 需用户确认 |
| 跨项目引用 | 需用户显式授权 |
| 归档整个项目 | 需用户确认 |

### 与专注区的关系

- 专注区的 `_本周.md` 中用 `[项目名]` 标签关联任务
- 项目区存放项目资料，专注区管理当周进度
- 周归档时，项目区的文件不动，只是专注区的任务标记完成

## AI 行为指南

### 新项目创建时
- 建议创建 README.md，包含：项目名称、目标、关键人物、时间节点
- 如果用户提供了项目背景，主动整理成结构化文档

### 项目工作中
- 在当前项目范围内主动搜索相关文件
- 发现可复用的模式时，提议记录到 MEMORY.md（程序性记忆）
- 不把项目 A 的细节带入项目 B 的对话

### 项目归档时
- 帮用户整理最终产出清单
- 确认是否有未完成事项
- 移入 `_已归档/` 后标记为只读

## 自定义建议

- 如果你的项目有统一的文件结构，可以在这里定义模板
- 如果某些项目有特殊的保密要求，在项目 README 中注明
- 可以按「客户项目 / 个人项目 / 合作项目」再分一层子目录
