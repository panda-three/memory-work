1 打印1-5（最简单的for循环）
生活类比
就像你在操场上跑步：
• 开始：从第1圈开始（i=1）
• 继续条件：只要没跑到第5圈就继续跑（i<=5）
• 每圈做的事：喊一声"我跑了第X圈！"（printf）
• 下一圈：圈数+1（i++）
// 目标：打印数字 1 2 3 4 5 for (int i = 1; i <= 5; i++) { printf("%d ", i); // 打印当前的i }
关键问题：循环会执行几次？每次i的值是多少？
执行次数 循环变量 i 条件判断 (i≤5) 执行动作 输出结果
初始化 i = 1 - 给i赋值为1 -
第1次 1 ✓ 真（1≤5） printf("%d ", 1) 1
第2次 2 ✓ 真（2≤5） printf("%d ", 2) 2
第3次 3 ✓ 真（3≤5） printf("%d ", 3) 3
第4次 4 ✓ 真（4≤5） printf("%d ", 4) 4
第5次 5 ✓ 真（5≤5） printf("%d ", 5) 5
第6次 6 ✗ 假（6≤5） 循环结束！ -
总结：循环执行了5次，输出：1 2 3 4 5
2 求1-5的和（累加问题）
生活类比
就像你在存钱罐里攒钱：
• 存钱罐：sum变量（开始是空的，sum=0）
• 每天：往里面放i元钱（sum = sum + i）
• 第1天：放1元，存钱罐有1元
• 第2天：放2元，存钱罐有3元
• 第3天：放3元，存钱罐有6元
• ...以此类推
// 目标：计算 1+2+3+4+5 的和 int sum = 0; // 存钱罐（开始是空的） for (int i = 1; i <= 5; i++) { sum = sum + i; // 把当前的i加到sum里 } printf("1+2+3+4+5 = %d\n", sum);
执行次数 循环变量 i sum的旧值 执行 sum=sum+i sum的新值
初始化 - 0 - 0
第1次 1 0 sum = 0 + 1 1
第2次 2 1 sum = 1 + 2 3
第3次 3 3 sum = 3 + 3 6
第4次 4 6 sum = 6 + 4 10
第5次 5 10 sum = 10 + 5 15
总结：最终sum = 15
核心思想：每次循环都把当前的i"攒"到sum里
3 统计满足条件的数字（计数问题）
生活类比
就像你在数学考试中数对号：
• 试卷：从第start题到第end题
• 计分器：count变量（开始是0分）
• 每道题：检查是否同时被6和8整除
• 答对了：计分器+1（count++）
• 答错了：跳过，继续下一题
// 目标：在范围内找既能被6又能被8整除的数 int start = 1, end = 100; int count = 0; // 计数器 for (int i = start; i <= end; i++) { if (i % 6 == 0 && i % 8 == 0) { // 同时满足两个条件 count++; // 找到一个，计数器+1 } }
关键问题：什么数既能被6整除，又能被8整除？
数学知识：这样的数一定能被6和8的最小公倍数整除！
• 6 = 2 × 3
• 8 = 2 × 2 × 2
• 最小公倍数 = 2³ × 3 = 24
循环变量 i i % 6 i % 8 满足条件？ count值
1~23 不满足条件，跳过... ✗ 0
24 0 ✓ 0 ✓ ✓ 找到了！ 1
25~47 不满足条件，跳过... ✗ 1
48 0 ✓ 0 ✓ ✓ 找到了！ 2
49~71 不满足条件，跳过... ✗ 2
72 0 ✓ 0 ✓ ✓ 找到了！ 3
73~95 不满足条件，跳过... ✗ 3
96 0 ✓ 0 ✓ ✓ 找到了！ 4
97~100 不满足条件，跳过... ✗ 4
答案：在1~100之间，有4个数字满足条件
它们是：24、48、72、96
规律：每隔24就有一个符合条件的数
💡 for循环执行流程图解
①初始化
int i=1
（只执行1次）
→
②判断条件
i≤5?
→
③执行循环体
printf(...)
→
④更新变量
i++
↺

执行完④后，回到②再次判断，形成循环

记忆口诀：
• 初始化只做一次（i=1）
• 判断条件决定继续还是结束（i≤5?）
• 条件为真，执行循环体（printf）
• 更新变量（i++）
• 回到判断，重复2-3-4步骤
⚠️ 学生常见错误
错误1：死循环
for (int i = 1; i <= 5; ) { // 忘记写 i++ printf("%d ", i); } // i永远是1，条件永远为真！
原因：循环变量没有更新，条件永远成立
错误2：循环一次都不执行
for (int i = 5; i < 1; i++) { // 5<1? 一开始就是假！ printf("%d ", i); }
原因：初始值就不满足条件
错误3：多执行或少执行一次
// 目标：打印1-5 for (int i = 1; i < 5; i++) { // 应该是 i<=5 printf("%d ", i); } // 只会打印 1 2 3 4，漏了5！
原因：条件判断符号用错了（< 和 ≤）
🔧 调试技巧：如何检查循环
技巧1：在循环里多打印信息
for (int i = 1; i <= 5; i++) { printf("第%d次循环，i=%d\n", i, i); // 看看每次i的值 }
技巧2：手工画trace table
在纸上画表格，模拟计算机执行过程：
• 第1列：执行次数
• 第2列：循环变量的值
• 第3列：其他重要变量（如sum、count）
• 第4列：条件判断结果
技巧3：检查这4个关键点
✓ 初始值对不对？（i应该从几开始？）
✓ 结束条件对不对？（用<还是≤？）
✓ 循环体里做的事对不对？
✓ 更新方式对不对？（i++、i--、i+=2？）

💪 循环学习三步走

第一步：理解执行流程（看trace table）
第二步：模仿例题，改改数字（从1-5改成1-10）
第三步：尝试新问题（打印偶数、求乘积等）
