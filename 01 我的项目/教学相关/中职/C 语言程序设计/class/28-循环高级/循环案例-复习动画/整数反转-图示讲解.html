🔄 整数反转详解

把数字"镜像翻转"的魔法

一、什么是整数反转？
定义：将一个整数的数字顺序颠倒
123 → 321
生活类比：
就像把扑克牌重新排列：
• 原始牌：1️⃣ 2️⃣ 3️⃣（从左到右）
• 反转后：3️⃣ 2️⃣ 1️⃣（从右到左）

或者像队列掉头：
• 原队伍：小明(1号) → 小红(2号) → 小刚(3号)
• 掉头后：小刚(3号) → 小红(2号) → 小明(1号)

二、核心思路（重要！）

🎯 反转的秘诀：从右往左，一个一个"挖"出来

步骤1：从原数字最右边"挖"出一位数字（用 % 10）
123 % 10 = 3 ← 挖出个位
步骤2：把挖出来的数字"贴"到新数字的最右边
result = 0 × 10 + 3 = 3
步骤3：把原数字去掉最右边一位（用 / 10）
123 / 10 = 12 ← 剩下的数字
步骤4：重复1-3步骤，直到原数字变成0
核心公式：
result = result × 10 + (num % 10)
每次都是：把旧结果×10（腾出一位），再加上新挖出的数字
三、详细执行过程（以123为例）
循环次数 num
（原数字）		 num % 10
（挖出的数字）		 result旧值 result×10
（腾出一位）		 result新值
（加上挖出的数字）		 num / 10
（去掉最右一位）
初始 123 - 0 - 0 -
第1次 123 3 0 0 × 10 = 0 0 + 3 = 3 12
第2次 12 2 3 3 × 10 = 30 30 + 2 = 32 1
第3次 1 1 32 32 × 10 = 320 320 + 1 = 321 0
结束 0 num变成0，循环结束 ✓ 321
关键观察：
• 每次循环，result都会"长大"一位
• 第1次：result = 3（一位数）
• 第2次：result = 32（两位数）
• 第3次：result = 321（三位数）
• 数字就是这样从右往左一个个"贴"上去的！
四、为什么要 × 10？
想象你在组装数字：

场景：你有一块写着"3"的木板，现在想在它右边加一个"2"

问题：怎么把"3"变成"32"？

答案：
① 把"3"往左移一位（相当于×10） → 变成"30"
② 在个位放上"2" → 30 + 2 = 32
第1次循环后：result = 3

五、完整代码实现
// 整数反转 #include <stdio.h> int main() { int num = 123; int result = 0; // 存放反转后的结果 printf("原数字：%d\n", num); printf("反转过程：\n"); // 只要num还有数字，就继续反转 while (num != 0) { int digit = num % 10; // ①挖出最右边的数字 printf(" 挖出数字%d，result从%d变成", digit, result); result = result * 10 + digit; // ②×10腾位 + 贴上新数字 printf("%d\n", result); num = num / 10; // ③去掉最右边的数字 } printf("\n反转后的数字：%d\n", result); return 0; }
代码运行结果：
原数字：123 反转过程： 挖出数字3，result从0变成3 挖出数字2，result从3变成32 挖出数字1，result从32变成321 反转后的数字：321

答案：反转后是1（不是0001）
原因：因为0001在整数中就是1，前导0会自动消失
七、学生常见错误
❌ 错误1：忘记×10
result = result + digit; // 错误！没有×10就无法"腾位"
后果：123反转后变成6（3+2+1），而不是321
❌ 错误2：忘记更新num
while (num != 0) { result = result * 10 + num % 10; // 忘记写 num = num / 10; }
后果：死循环！num永远不变，条件永远为真
❌ 错误3：循环条件写错
while (num > 0) { // 如果num是负数，一次都不执行
建议：用 num != 0 更安全（可以处理负数）

🎯 核心要点

① 反转 = 从右往左一个个"挖"出来再"贴"上去
② 挖数字：num % 10
③ 贴数字：result = result × 10 + digit
④ 去掉已处理的：num = num / 10
