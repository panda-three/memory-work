                            【使用流程】
================================================================================

第一步：布置作业（5分钟）
  1. 将"作业任务书.txt"发给学生（可打印或发电子版）
  2. 在班级群或课上讲解作业要求
  3. 强调以下几点：
     ✓ 必须独立完成，可以讨论思路但不能抄代码
     ✓ 提交截止时间
     ✓ 提交方式（源代码+截图+说明）
     ✓ 抄袭后果
  4. 询问学生是否有疑问

第二步：中途检查（可选，第3-4天）
  1. 课上抽15分钟让学生展示进度
  2. 解答学生遇到的问题
  3. 给学困生一些提示（可以让他们参考基础版代码）

第三步：批改作业（截止后）
  1. 使用"评分标准和检查清单.txt"进行批改
  2. 对于可疑作业（可能抄袭），进行答辩
  3. 记录每个学生的得分和评语

第四步：成果展示（批改后下节课）
  1. 选3-5个优秀作品让学生上台演示
  2. 点评优秀作品的亮点
  3. 总结大家的常见问题
  4. 给优秀学生奖励（平时分+5，或小礼品）

================================================================================
                          【参考代码使用说明】
================================================================================

⚠️ 重要提醒：
参考代码仅供老师和学困生参考，不要直接发给全班！
如果全班都有参考代码，很多学生会直接抄，失去作业的意义。

建议使用方式：

方式一：只给学困生看基础版
  - 适合对象：编程基础很差，完全无法下手的学生
  - 给予方式：私下发给他们，或让他们来办公室看
  - 要求：看懂后用自己的方式重写，不能直接复制

方式二：作为批改参考
  - 老师批改作业时，参考各个难度版本的代码
  - 判断学生的完成度和创新度
  - 确定合理的分数档次

方式三：课后公开（作业提交后）
  - 等所有学生提交作业后，再公开参考代码
  - 让学生对比自己的代码和参考代码
  - 讲解不同难度版本的差异

================================================================================
                          【三个难度版本说明】
================================================================================

基础版（60分水平）
├─ 特点：
│  - 功能简单，只实现核心功能（2-3个）
│  - 代码量少（80-120行）
│  - 适合编程基础较差的学生
│  - 没有输入验证，没有美化
│
├─ 函数数量：3-4个
│
└─ 示例：
   家庭账本只有收入、支出、查询余额
   成绩管理只能录入和查询

标准版（80分水平）
├─ 特点：
│  - 功能完整，实现题目要求的所有主要功能
│  - 代码量中等（150-250行）
│  - 有基本的输入验证
│  - 有完整的注释
│
├─ 函数数量：5-7个
│
└─ 示例：
   家庭账本有收入、支出、余额查询、显示所有记录
   成绩管理有录入、查询、平均分、最高/低分、及格率

进阶版（100分水平）
├─ 特点：
│  - 功能丰富，有创新点
│  - 代码量较大（300-500行）
│  - 有详细的输入验证和错误处理
│  - 界面美观（使用框线、符号美化）
│  - 有额外功能（如历史记录、统计分析等）
│
├─ 函数数量：8-10个
│
└─ 示例：
   家庭账本有统计功能、删除记录功能、余额预警
   成绩管理有排序、按姓名查询、删除记录

                          【常见问题解答】
================================================================================

Q1: 作业时间1周够吗？会不会太长或太短？
A1: 1周比较合适。时间太短（3-4天）很多学生来不及；时间太长（2周）学生容易拖延。
    如果班级整体水平较好，可以缩短到5天；如果较差，可以延长到10天。

Q2: 如果学生完全不会做怎么办？
A2: 第一次课后作业，完全不会是正常的。可以：
    - 课上多给15-20分钟让学生开始写，当场解答问题
    - 建立作业答疑群，鼓励学生互相讨论（但不能发代码）
    - 对学困生可以单独辅导，给他们看基础版代码

Q3: 需要准备多少份参考代码？
A3: 本次已准备4个题目×3个难度=12份代码，足够使用。
    如果学生自创题目，可以根据情况额外准备1-2份参考。

Q4: 如何防止学生抄袭？
A4: 完全防止很难，但可以降低抄袭率：
    - 提前说明抄袭后果（答辩+重做+扣分）
    - 参考代码不要一开始就全部公开
    - 批改时注意代码相似度
    - 对于高分作业，随机抽几个答辩

Q5: 批改一个班（40人）的作业大概需要多长时间？
A5: 如果使用快速检查清单，平均每人2-3分钟，40人大约2小时。
    第一次批改会慢一些，熟练后可以控制在1.5小时以内。

Q6: 作业成绩占期末成绩的多少合适？
A6: 建议占10-15%。如果这是唯一的课后大作业，可以占20%。

Q7: 学生可以多选几个题目来做吗？
A7: 可以鼓励学有余力的学生多做，作为加分项。
    每多做一个题目（且质量达标），可以加5-10分。

Q8: 作业需要打印出来吗？
A8: 不需要。电子版提交即可，既环保又方便批改。
    可以要求学生把代码、截图、说明打包成一个文件夹，命名为"学号-姓名"。

                          【成果展示建议】
================================================================================

作业批改完成后，建议在课上展示优秀作品，具体流程：

1. 开场（2分钟）
   "这次作业大家完成得不错，全班平均分XX分。
    今天我们请几位同学来展示他们的作品。"

2. 学生展示（每人5分钟，共3-5人）
   - 学生上台用投影演示程序
   - 讲解自己的实现思路
   - 分享遇到的困难和解决方法
   - 其他同学可以提问

3. 老师点评（3分钟/人）
   - 指出亮点（功能创新、界面美观、代码规范等）
   - 给出改进建议
   - 其他同学可以学习的地方

4. 问题总结（10分钟）
   "这次作业大家普遍存在以下问题：
    1. XXX（如：函数数量不够）
    2. XXX（如：没有输入验证）
    3. XXX（如：代码注释不完整）"

5. 公开参考代码（5分钟）
   - 展示基础版、标准版、进阶版的区别
   - 让学生对比自己的代码
   - 讲解关键技术点

6. 鼓励和展望（2分钟）
   "这次作业是大家第一次完成完整的程序，做得很好！
    接下来我们要学习数组，到时候可以做更复杂的程序。"
