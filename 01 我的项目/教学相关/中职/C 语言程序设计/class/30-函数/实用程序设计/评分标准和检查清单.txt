================================================================================
                    C语言函数综合作业 - 评分标准和检查清单
================================================================================

【快速评分流程】

第一步：快速检查（30秒/人）
  ✓ 查看运行截图 → 能运行？有功能？
  ✓ 打开代码文件 → 有函数定义？代码整洁？
  ✓ 阅读说明文档 → 写清楚了吗？

第二步：详细评分（2-3分钟/人）
  ✓ 按照下面的评分表逐项打分
  ✓ 记录亮点和问题
  ✓ 确定最终分数

第三步：答辩（可选，针对可疑作业）
  ✓ 让学生解释某段代码的作用
  ✓ 让学生演示程序运行
  ✓ 询问实现思路

================================================================================
                              【评分标准】
================================================================================

┌──────────────────────────────────────────────────────────────────┐
│ 一、程序可运行性（30分）                                          │
├──────────────────────────────────────────────────────────────────┤
│ 30分：无编译错误，所有功能正常运行，无明显bug                     │
│ 25分：能运行，主要功能正常，有1-2个小bug                          │
│ 20分：能运行，但部分功能有问题                                    │
│ 15分：勉强能运行，很多功能不正常                                  │
│ 10分：有编译错误，但稍作修改可以运行                              │
│  0分：无法运行，或只有框架没有实现                                │
│                                                                   │
│ 检查要点：                                                        │
│  □ 编译是否通过？                                                 │
│  □ 运行截图是否真实？                                             │
│  □ 菜单选择是否正常？                                             │
│  □ 输入错误数据是否会崩溃？                                       │
└──────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────┐
│ 二、函数使用（30分）                                              │
├──────────────────────────────────────────────────────────────────┤
│ 30分：至少3个函数，调用合理，参数和返回值使用正确                 │
│ 25分：至少3个函数，调用基本合理，有小瑕疵                         │
│ 20分：有3个函数，但调用不够合理（如main函数代码过多）             │
│ 15分：有2个函数，未达到要求                                       │
│ 10分：只有1个函数，或函数使用不当                                 │
│  0分：没有自定义函数，所有代码都在main函数里                      │
│                                                                   │
│ 检查要点：                                                        │
│  □ 函数数量是否达到3个以上？                                      │
│  □ 每个函数是否有明确的功能？                                     │
│  □ main函数是否只负责调度？                                       │
│  □ 函数命名是否有意义？                                           │
│  □ 参数和返回值使用是否合理？                                     │
└──────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────┐
│ 三、功能完整度（20分）                                            │
├──────────────────────────────────────────────────────────────────┤
│ 20分：实现了题目要求的所有主要功能，且功能完善                    │
│ 16分：实现了大部分功能（80%以上）                                 │
│ 12分：实现了基本功能（60%左右）                                   │
│  8分：功能不完整（40%左右）                                       │
│  4分：只实现了极少功能                                            │
│  0分：基本没有功能实现                                            │
│                                                                   │
│ 检查要点（根据题目）：                                            │
│                                                                   │
│ 家庭账本：                                                        │
│  □ 记录收入  □ 记录支出  □ 查询余额  □ 显示记录  □ 循环菜单      │
│                                                                   │
│ 成绩管理器：                                                      │
│  □ 录入成绩  □ 查询成绩  □ 计算平均分  □ 最高/低分  □ 及格率     │
│                                                                   │
│ 单位换算器：                                                      │
│  □ 长度换算  □ 重量换算  □ 温度换算  □ 时间换算  □ 至少4类       │
│                                                                   │
│ ATM系统：                                                         │
│  □ 密码验证  □ 查询余额  □ 取款  □ 存款  □ 修改密码              │
└──────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────┐
│ 四、代码规范（10分）                                              │
├──────────────────────────────────────────────────────────────────┤
│ 10分：代码整洁，注释完整，缩进正确，命名规范                      │
│  8分：基本规范，注释较完整，有小瑕疵                              │
│  6分：注释不够完整，或缩进有问题                                  │
│  4分：代码较混乱，注释很少                                        │
│  2分：几乎没有注释，代码很乱                                      │
│  0分：完全不规范                                                  │
│                                                                   │
│ 检查要点：                                                        │
│  □ 每个函数都有注释说明吗？                                       │
│  □ 缩进是否统一？（全部用tab或全部用空格）                        │
│  □ 变量命名是否有意义？（不要用a、b、c这种）                      │
│  □ 代码是否有空行分隔逻辑块？                                     │
│  □ 括号位置是否统一？                                             │
└──────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────┐
│ 五、创新/实用性（10分，加分项）                                   │
├──────────────────────────────────────────────────────────────────┤
│ 10分：有明显创新，或实用性很强，或有亮点功能                      │
│  8分：有一定创新，界面美观或功能丰富                              │
│  6分：有小创新，如输入验证、错误提示等                            │
│  4分：基本完成要求，略有亮点                                      │
│  2分：中规中矩，无特色                                            │
│  0分：完全照搬参考代码                                            │
│                                                                   │
│ 创新点示例：                                                      │
│  ✓ 界面美观（使用框线、颜色等）                                   │
│  ✓ 有数据验证（防止用户输入错误）                                 │
│  ✓ 有统计分析功能                                                 │
│  ✓ 有历史记录功能                                                 │
│  ✓ 自创题目且实用                                                 │
│  ✓ 代码优化（如使用switch代替多个if-else）                        │
└──────────────────────────────────────────────────────────────────┘
