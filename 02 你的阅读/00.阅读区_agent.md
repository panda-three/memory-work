# 阅读区代理

## 区域概述

**你的阅读**是你的输入端——书籍笔记、文章摘要、论文批注、视频总结、播客记录，所有从外部吸收的知识都沉淀在这里。

这个区域的核心逻辑是**知识沉淀**：不是剪藏链接的收藏夹，而是经过你思考加工的知识库。原始素材可以存，但更有价值的是你的批注、摘要和洞见。

---

## 区域结构

```
02 你的阅读/
├── 00.阅读区_agent.md        ← 本文件（区域规则）
├── 书籍/                     ← 读书笔记
├── 文章/                     ← 文章摘要与批注
├── 论文/                     ← 学术论文笔记
├── 视频与播客/               ← 音视频内容总结
├── 关于我/                   ← 个人档案（高敏感）
│   ├── 核心档案.md
│   ├── 背景经历.md
│   └── 能力图谱.md
└── [其他分类]/
```

> 分类方式可自定义。按来源分（书/文/视频）、按主题分（技术/商业/创意）、或两者结合都行。

## 区域规则

### 敏感度分级

| 子目录 | 敏感度 | 修改规则 |
|--------|--------|----------|
| `关于我/` | **高** | 任何修改需**逐条确认** |
| 其他目录 | 低 | 正常操作 |

### 笔记规范（建议，非强制）

每份阅读笔记建议包含：

```markdown
---
title: [标题]
type: [书籍/文章/论文/视频/播客]
source: [来源链接或出处]
date_read: YYYY-MM-DD
tags: [标签]
---

## 核心观点
[用自己的话概括，不是复制粘贴]

## 关键摘录
[值得反复看的原文片段]

## 我的思考
[你的批注、联想、质疑、启发]

## 行动项（可选）
[读完后想做的事]
```

### 操作规范

| 操作 | 权限 |
|------|------|
| 读取 | 自由，AI 按需检索 |
| 新增笔记 | 自由 |
| 修改已有笔记 | 追加批注可直接操作；修改原有内容需说明 |
| 删除 | 需用户确认 |
| `关于我/` 修改 | 逐条确认 |

### 与其他区域的关系

- **专注区**：当前周的阅读任务在 `_本周.md` 中追踪，完成后笔记存入此区域
- **写作区**：阅读笔记是写作的素材来源，AI 在写作时可主动检索此区域
- **MEMORY.md**：阅读中发现的稳定偏好或认知模式，可提议写入长期记忆

## AI 行为指南

### 用户分享阅读内容时
- 帮助提取核心观点和结构
- 问「你觉得哪部分最有启发」而非直接替用户总结
- 如果用户提供视频链接，可调用 bibigpt-summary 技能提取逐字稿

### 整理笔记时
- 按用户习惯的格式整理（观察前几份笔记的风格）
- 发现与已有笔记的关联时，主动提示（「这和你上次读的 XX 有关联」）
- 不擅自删除用户的原始批注

### 检索素材时
- 被写作区或项目区引用时，提供相关笔记的摘要
- 引用时标注来源（哪份笔记、哪个段落）

---

## 自定义建议

- 分类方式按你的思维习惯来，不用照搬模板
- 如果你有高频阅读的领域，可以建专门的子目录
- 定期回顾旧笔记——阅读区不是写完就不看的归档区
