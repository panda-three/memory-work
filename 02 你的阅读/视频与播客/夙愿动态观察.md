## 核心观点

### 工具链分层（夙愿的建议顺序）

1. **Cherry Studio + Claude 模型** — 先把模型本身玩明白
2. **Claude Code** — 用 AI 赋能日常工作
3. **OpenClaw（小龙虾）** — 满足各种自动化需求

顺序不能倒，否则会很痛苦，而且不知道自己为什么痛苦。

### 无 SOP，不龙虾

先把流程想清楚、跑通了，再交给自动化工具执行。自动化的前提是你已经有了清晰的 SOP。

### 工具关系

- Claude Code = 开发工具
- OpenClaw = 自动化产品
- 用 Claude Code 不断增强 OpenClaw 的能力，形成飞轮

## 我的思考

- 这和李一舟「业务自营」是同一个逻辑：先分清哪些必须自己想明白（SOP 设计），哪些可以交出去（自动化执行）
- 我当前处在第 2 步（Claude Code 赋能日常工作），记忆系统本身就是实践
- SOP 积累够了再上 OpenClaw，顺序是对的
