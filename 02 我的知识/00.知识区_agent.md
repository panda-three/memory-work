# 知识区代理

## 区域概述

**我的知识**是你的知识沉淀中心——被动阅读的笔记、主动攻克的研究、验证过的最佳实践，所有经过你思考加工的知识都按主题组织在这里。

这个区域的核心逻辑是**按主题聚合**：不按获取方式分（被动/主动），不按内容类型分（笔记/实践），而是按你关心的领域分。想找公众号相关的东西，进一个文件夹全部找齐。

---

## 区域结构

```
02 我的知识/
├── 00.知识区_agent.md        ← 本文件（区域规则）
├── 关于我/                   ← 个人档案（高敏感）
├── [主题名]/                 ← 按主题自然生长
│   ├── _index.md             ← 可选：本主题最佳实践速查
│   ├── xxx笔记.md            ← type: note
│   ├── xxx最佳实践.md        ← type: best-practice
│   ├── xxx探索记录.md        ← type: exploration
│   └── 学习-xxx/             ← type: learning（交互式学习序列，由技能触发）
│       ├── 01.md
│       └── 02.md
└── [新主题]/                 ← 新主题出现时创建，不预设空目录
```

> 主题目录按需创建，不提前铺设。一个领域积累到第一份文件时自然建目录。

## 区域规则

### 敏感度分级

| 子目录 | 敏感度 | 修改规则 |
|--------|--------|----------|
| `关于我/` | **高** | 任何修改需**逐条确认** |
| 其他目录 | 低 | 正常操作 |

### 文件类型与 frontmatter

每份文件使用以下 frontmatter：

```yaml
---
title: [标题]
type: [note / best-practice / exploration]
verified: true/false        # best-practice 专用
source: [来源：自己验证 / 课程 / 对标对象 / 链接]
created: YYYY-MM-DD
tags: [标签]
---
```

三种文件类型：

| type | 说明 | 标记 |
|------|------|------|
| note | 阅读笔记、课程总结、认知记录 | 默认类型 |
| best-practice | 验证过的、可复用的方法 | `verified: true` |
| exploration | 试过但没走通，保留上下文 | `verified: false` |

### 笔记规范（建议，非强制）

每份笔记建议包含：

```markdown
## 核心观点
[用自己的话概括，不是复制粘贴]

## 关键摘录
[值得反复看的原文片段]

## 我的思考
[你的批注、联想、质疑、启发]

## 行动项（可选）
[读完后想做的事]
```

### 操作规范

| 操作 | 权限 |
|------|------|
| 读取 | 自由，AI 按需检索 |
| 新增笔记 | 自由 |
| 修改已有笔记 | 追加批注可直接操作；修改原有内容需说明 |
| 删除 | 需用户确认 |
| `关于我/` 修改 | 逐条确认 |

## 索引机制

每个主题目录可选一个 `_index.md`，作为该主题的**最佳实践速查**。

```markdown
---
title: [主题名] · 速查索引
type: index
updated: YYYY-MM-DD
---

# [主题名] · 当前最佳做法

- **排版**：使用 XXX 方案（→ 排版最佳实践.md）
- **选题**：按 XXX 逻辑筛选（→ 选题方法.md）
```

规则：
- 文件少的主题不需要 `_index.md`，文件多了再加
- AI 检索某主题时优先读 `_index.md`，快速定位靠谱答案
- 最佳实践更新时同步更新索引

## 验证升级

一条知识从 note / exploration 升级为 best-practice 的门槛：

至少满足其一：
- 自己实践过至少一次，确认有效
- 从可信来源提炼，经 Panda 确认认可

升级时：
1. 更新文件 frontmatter：`type: best-practice`，`verified: true`
2. 如该主题有 `_index.md`，同步更新索引

探索记录（exploration）的价值：保留上下文，避免下次从零开始。条件成熟时回来继续推进，验证通过后升级为 best-practice。

## AI 行为指南

### 用户分享阅读内容时
- 帮助提取核心观点和结构
- 问「你觉得哪部分最有启发」而非直接替用户总结
- 判断主题归属，放入对应主题目录

### 整理笔记时
- 按用户习惯的格式整理（观察前几份笔记的风格）
- 发现与已有笔记的关联时，主动提示（「这和你之前在 XX 主题下记的有关联」）
- 不擅自删除用户的原始批注

### 检索素材时
- 被写作区或项目区引用时，先读 `_index.md`（如有），再深入具体文件
- 引用时标注来源（哪个主题、哪份笔记、哪个段落）

### 与其他区域的关系

- **专注区**：当前周的学习任务在 `_本周.md` 中追踪，完成后笔记存入此区域
- **写作区**：知识笔记是写作的素材来源，AI 在写作时可主动检索此区域
- **工具区**：交互式学习流程由 `06 我的工具/interactive-learning/` 技能驱动，产出文件存入对应主题目录
- **MEMORY.md**：阅读或研究中发现的稳定偏好或认知模式，可提议写入长期记忆

## 自定义建议

- 主题目录按你的关注领域自然生长，不用照搬模板
- 如果某个主题下文件超过 10 个，考虑加 `_index.md` 做速查
- 定期回顾旧笔记——知识区不是写完就不看的归档区
