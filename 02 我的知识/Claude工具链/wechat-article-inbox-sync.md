# 灵感总是丢？我搭了一套系统，手机说一句话就存进知识库

## 一个让我焦虑了很久的问题

我不知道你有没有这种感觉：脑子里经常冒出一些想法，有时候是刷到某个内容触发的，有时候是走在路上突然想到的。你觉得这个想法挺重要的，不想丢。

然后你打开微信，发给文件传输助手。

问题是，然后呢？

我之前就是这样。所有灵感都往微信文件助手里扔。但微信不是干这个的——你的想法和工作群消息、外卖通知、家人聊天混在一起。等你真正想用的时候，要么翻不到，要么翻到了已经想不起来当时为什么觉得它重要。

大量的想法就这么被浪费了。说实话，这件事让我非常焦虑。

不只是灵感的问题。我没有一个统一的地方来管理我的知识。学到的东西散落在各种 App、各种聊天记录、各种收藏夹里，从来没有被系统地整理过。我的知识管理完全没有章法。

## 转折点

直到我看到小七姐做的一套知识管理系统。

她的思路让我眼前一亮：**不是你自己去整理知识，而是让 AI 帮你整理。** 你只管往里面扔东西，AI 负责分类、归档、追踪。

我想，这不就是我需要的吗？

于是我开始动手搭自己的版本。

## 踩了几个坑

搭的过程没有一帆风顺。

**第一个坑：Working Copy。** 这是 iOS 上一个 Git 客户端，可以在手机上直接操作代码仓库。听起来很完美，但打开一看——198 元。我只是想在手机上记个灵感，花 198 买个 Git 客户端，有点杀鸡用牛刀了。

**第二个坑：GitHub 官方 App。** GitHub 自己出了手机 App，可以直接在仓库里创建文件。但我在大陆，App Store 里下不了。

两条路都走不通。

## 最终方案：零成本，一步到位

折腾了一圈之后，我找到了一个既免费又好用的方案：**用一个网页书签，通过 GitHub API 直接把内容写进仓库。**

具体是这样的：

我在知识库仓库里放了一个 HTML 页面，部署到 GitHub Pages 上。然后在 iPhone 上用 Safari 打开这个页面，添加到主屏幕。

从此，我的主屏幕上多了一个"灵感捕捉"的图标。

**现在我的流程是：**

点图标 → 页面弹出来 → 语音输入（或者打字）→ 点"发射" → 完成。

想法直接变成一个 Markdown 文件，存进我知识库的收件箱目录。整个过程不超过 10 秒。不需要打开微信，不需要复制粘贴，不需要任何中间步骤。

**而且完全免费。** GitHub 仓库免费，GitHub Pages 免费，GitHub API 免费。唯一需要的就是一个 GitHub 账号和一次初始配置。

## 但光存进去还不够

如果只是换了个地方存，收件箱最终也会变成另一个垃圾堆。

所以我给 AI 助手加了一个规则：**每次启动，自动扫描收件箱。**

AI 会读取我扔进去的每条内容，帮我判断——这条是产品灵感？学习笔记？还是对某个问题的思考？然后告诉我它的分类建议，我确认后，AI 把内容搬到知识库里对应的位置，清空收件箱。

**我只管往箱子里扔，AI 帮我收拾。**

这个流程能跑通的前提是：AI 了解我的整个知识库结构。它知道哪个文件夹放什么，知道我最近在关注什么，知道我的工作重点是什么。所以它的分类不是简单的关键词匹配，而是基于对我整个上下文的理解。

## 整套系统长什么样

说说全貌。我的知识库就是一个文件夹，里面全是 Markdown 文件，用 Git 做版本管理：

```
知识库/
├── 00 Focus Zone/    ← 本周工作区（任务、进度、产出）
├── 01 Materials/     ← 参考资料
├── 03 Projects/      ← 项目空间
├── 07 Inbox/         ← 收件箱（手机灵感都进这里）
├── USER.md           ← 我的个人档案和偏好
├── MEMORY.md         ← AI 的长期记忆
└── SOUL.md           ← AI 的协作风格定义
```

每个区域各司其职。AI 每次启动会读取我的档案和记忆，知道我是谁、在做什么、关注什么。然后扫描工作区和收件箱，帮我同步状态。

为什么用 Markdown + Git？因为简单、通用、免费。Markdown 任何编辑器都能打开，Git 让每次修改都有记录。不依赖任何特定 App，不怕哪天某个服务停了数据全丢。

## 如果你也想搭一套

整个过程分五步：

**第 1 步：创建 GitHub 仓库**

把你的知识库文件夹推到 GitHub 上。如果你还没有知识库，先建一个文件夹，里面放几个 Markdown 文件就行。

```bash
git init
git remote add origin https://github.com/你的用户名/你的仓库名.git
git add -A
git commit -m "init: knowledge base"
git push -u origin main
```

**第 2 步：生成 GitHub Token**

1. 打开 https://github.com/settings/tokens?type=beta
2. 点 "Generate new token"
3. Token name 随便起，比如 `inbox-capture`
4. Repository access 选 "Only select repositories"，选你的知识库仓库
5. Permissions → Contents 设为 Read and write
6. 生成后复制 token 保存好（只显示一次）

**第 3 步：创建捕捉页面**

在仓库的 `07 Inbox/` 目录下创建一个 `capture.html`。核心逻辑就是一个输入框 + 一个按钮，点击后通过 GitHub API 把内容创建为一个 `.md` 文件。

关键代码就这几行：

```javascript
fetch('https://api.github.com/repos/你的仓库/contents/07 Inbox/' + filename, {
  method: 'PUT',
  headers: {
    'Authorization': 'Bearer 你的token',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    message: 'inbox: quick capture',
    content: btoa(你的输入内容)
  })
});
```

我做了一个完整版的页面，带深色主题和状态反馈，源码开源在我的仓库里，可以直接拿去用。

**第 4 步：开启 GitHub Pages**

进入仓库 Settings → Pages → Source 选 "Deploy from a branch" → 选 main 分支 → 保存。

等几分钟，你的捕捉页面就能通过网址访问了。

**第 5 步：添加到手机主屏幕**

iPhone 用 Safari 打开页面 → 首次粘贴 Token → 点分享 → "添加到主屏幕"。

搞定。从此你的主屏幕上就有了一个灵感捕捉入口。

## 用了一段时间的感受

说实话，顺畅了很多。

以前灵感来了，我的第一反应是"算了，回头再说"——因为我知道存到微信里也大概率会丢。现在不一样了，掏出手机说两句话就完事，我知道它会被好好保管。

焦虑感少了很多。不是因为我变得更有条理了，而是因为有个系统在帮我兜底。

这大概就是工具的意义：不是让你变成一个更自律的人，而是让你不需要那么自律也能把事情管好。
