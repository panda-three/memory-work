# 提示词生成和调教指南案例展示

一条 prompt 从"正确但没用"到"精准命中"的完整迭代过程。基于小七姐提示词方法论，实战验证三个版本，记录每次失败的原因和修正逻辑。

---

## 背景

Panda 的卡点：不知道做什么产品。手里的牌是中职计算机教师身份 + AI 编程能力，目标是找到一个能从副业做起、最终通向财务自由的产品方向。

任务：写一条 prompt，让 AI 帮忙发现「需求缝隙」。

---

## 第一性原理

> **你的认知上限，是大模型的下限。**

prompt 是你认知的外显。你对问题想得多清楚，模型就能给你多精准的回答。模型的默认输出是"平庸解"——正确、安全、没意思。要跳出平庸曲线，你必须在 prompt 里暴露出你真正的认知。

---

## 设计框架：三种美

来自小七姐在有知有行的分享，三条 prompt 对应三种写法范式：

| 类型 | 核心机制 | 适用场景 | 代表技法 |
|------|----------|----------|----------|
| 结构之美 | 给模型思考的脚手架，控制处理信息的顺序 | 结构化任务、信息提取 | 分层框架、五钥标注、输出格式预定义 |
| 语言之美 | 用语言编排把认知结构"投影"到模型语义空间 | 创意生成、深度思考 | 多维约束叠加、正向标准+反向排除、行为约束 |
| 想象之美 | 给方向但留空间，构建让思维涌现的场域 | 元认知、开放探索 | 隐喻、留白、场域构建 |

好的 prompt 通常融合三种美，而非只用一种。

## 迭代全过程

### v1：出海硬约束

**prompt 核心设定**：

```
【硬约束】
- 时间结构：碎片时间推进 + 假期集中冲刺
- 能力杠杆：AI 编程是核心武器
- 冷启动：一个人跑通 MVP
- 出海适配：面向海外用户，或至少不被中文市场锁死
```

**输出结果**：三个纯海外场景——VBA 遗留系统迁移、硬件产品交互式说明书、程序化 SEO 数据清洁工。

**问题诊断**：输出质量不差，逻辑链完整，但离用户体感太远。Panda 对海外小企业的痛点没有真实经验，无法判断这些方向是不是真的。

**教训**：

> 约束决定了模型的搜索空间。把不熟悉的领域设为硬约束，等于逼模型在你没有体感的地方找答案。输出再"正确"你也接不住。

---

### v2：场景认知约束

**修改动作**：
- 「出海适配」从硬约束降级为软约束（加分项）
- 新增「场景认知」硬约束：优先从教育、职教、技能培训等有体感的领域找缝隙
- 排除清单加入「我没有体感的纯海外场景」

**prompt 核心设定**：

```
【硬约束】
- 场景认知：优先从我有真实体感的领域找缝隙——教育、职业教育生态、技能培训
- 时间结构 / 能力杠杆 / 冷启动（同 v1）

【软约束】
- 出海潜力：不强求，加分项
```

**输出结果**：三个职教场景——技能大赛代码审计工具、非计算机专业的交互式课件组件、跨境电商专业的客诉模拟训练。

**问题诊断**：场景落在了职教领域，方向对了，但仍然不是 Panda 的日常。技能大赛集训、信息化教学大赛、跨境电商实训——这些是"中职教师"这个标签的统计平均值，不是 Panda 每天真正在做的事。

**教训**：

> 标签不等于上下文。"中职计算机教师"六个字的信息量太低，模型只能去想象一个"平均的中职教师"在做什么。你需要给的不是职业名称，而是你具体的一天。

---

### v3：真实日常口述

**修改动作**：
- 去掉标签式描述（"中职计算机教师，带学生、懂教学设计"）
- 换成真实日常口述：备课、上课、职教高考、方法论沉淀的渴望
- 硬约束新增「从自用到产品」路径
- 排除清单加入 v2 跑偏的具体场景
- 品质锚点从"看到陌生人群"改为"看到自己日常里被忽视的东西"

**prompt 核心设定**：

```
他的日常是这样的：
每天备课、上课，教中职计算机。学生要参加职教高考，他最在意的是
自己的备课和讲课质量，能不能真正帮到学生的考试成绩。他最烦的是
备课和讲课中大量的重复劳动——备课应该怎么备？有哪些工作流？
讲课分哪些场景，每种场景怎么讲？这些东西他每天都在做，但从来
没有沉淀成一套可复用的方法论。

【硬约束】
- 从他的真实日常出发：备课、上课、职教高考备考、教学方法论沉淀
- 从自用到产品：先解决自己的问题，再看同类人群是否有同样的痛
- 时间结构 / 能力杠杆 / 冷启动（同前）

【品质锚点】
我要的是：读完之后，我能看到自己日常里一个一直在做但从没想过
可以变成产品的事情，而 AI 刚好能让它从"个人经验"变成"可复制的系统"。
```

**输出结果**：

| 缝隙 | 核心痛点 | 产品形态 |
|------|----------|----------|
| 备课的「考点-教法」匹配 | 备课凭经验和直觉，不知道知识点在高考里怎么考、学生错在哪 | 考频热力图 + 易错点分析 + 教法建议 |
| 讲课的「场景决策树」 | 学生卡住时的应对全靠即兴，经验碎片化无法复用 | 教学场景决策树生成器，把教学直觉变成教学系统 |
| 批改的「错因归类」 | 知道错的人多，但不知道错因是否相同，一刀切重讲对谁都没用 | 错因归类引擎，从批改者变成诊断者 |

**效果评估**：三个缝隙全部长在每天真正在做的事情上。痛的人就是自己，不需要想象陌生人群。三个方向还能串成一个完整链路：考点热力图（备课输入）→ 场景决策树（教学过程）→ 错因归类（教学反馈）。

## 迭代规律总结

### 三次失败的根因是同一个

prompt 里的约束和上下文，没有对齐用户的真实认知边界。

```
v1：约束指向没体感的领域 → 输出正确但接不住
v2：标签代替了上下文 → 输出落在统计平均值上
v3：用日常口述替代标签 → 输出精准命中
```

### 核心原则

1. **约束是认知匹配问题，不只是逻辑问题**。模型的搜索空间由你的约束决定。好的约束把模型引向你能判断真伪的区域。

2. **标签 ≠ 上下文**。"中职计算机教师"是标签，"每天备课上课、最烦重复劳动、想沉淀方法论"是上下文。模型需要后者。

3. **品质锚点要跟着场景走**。v1/v2 的锚点是"看到陌生人群的痛点"，v3 改成"看到自己日常里被忽视的东西"。锚点错了，模型的搜索方向就错。

4. **排除清单是活的**。每次跑偏的具体场景都应该加入排除清单，防止模型重蹈覆辙。

5. **从自用到产品**是独立开发者最短的验证路径。先解决自己的问题，再看同类人群是否有同样的痛。

### 提示词调教工作流

```
写初版 → 跑一遍 → 诊断输出偏差 → 定位是哪个约束/上下文出了问题
→ 修改对应区块 → 把跑偏场景加入排除清单 → 再跑 → 循环
```

关键：每次只改一个变量，才能定位问题。v1→v2 改了约束层级，v2→v3 改了上下文粒度。如果同时改两个，你不知道是哪个起了作用。

---

## 适用场景

这套迭代方法不限于"找产品方向"。任何需要 AI 在你的认知边界内搜索答案的场景都适用：

- 写商业计划时，让 AI 帮你找盲区
- 做技术选型时，让 AI 在你熟悉的栈里推荐方案
- 做内容选题时，让 AI 从你的经验里挖掘角度

核心不变：**你给的上下文越接近你的真实认知和体感，输出越有用。**
