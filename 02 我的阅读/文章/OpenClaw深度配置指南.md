# OpenClaw深度配置指南：打造专属AI同事的完整方案

核心定位：将 OpenClaw 视为「远程AI同事」，通过「让AI配置AI」实现全流程自动化。

---

## 一、准备工作：硬件与大脑部署

### 云服务器选择

| 关键决策 | 推荐方案 | 原因 |
|----------|----------|------|
| 地域 | 海外（香港/韩国/新加坡） | 避免代理配置复杂性 |
| 配置 | 内存≥4G（推荐8G/16G） | 2G会卡顿，浏览器功能需至少4G |
| 渠道 | 阿里云/腾讯云/火山引擎海外地域 | 简化注册流程 |

### 大模型选择

| 类型 | 代表 | 适用场景 |
|------|------|----------|
| 高端 | Claude Opus 4.6 | 复杂任务、深度分析 |
| 性价比 | GPT-5.3 Codex（ChatGPT Team） | 日常对话、非文科任务 |
| 国产平替 | GLM 5、Kimi K2.5、MiniMax M2.5 | 综合场景、本地化需求 |

### 安装部署

- 预装镜像：推荐云平台一键部署镜像
- 空白服务器：通过 Claude Code 自动安装
- 必备工具：Claude Code（精准修改配置）+ HappyCoder（手机远程操作）

## 二、网络能力配置

### 搜索：Brave Search

- 免费额度每月2000次，需绑定海外Visa卡
- 替代：闲鱼购买 API（几十元）

### 网页抓取：jina.ai Reader

- 免费、无需API key
- 绕过付费墙、抓取推特推文、返回Markdown格式
- 用法：`https://r.jina.ai/<目标URL>`

### 浏览器能力层级

| 层级 | 功能 | 场景 |
|------|------|------|
| L0 | 搜索+抓取（Brave + jina） | 80%日常信息获取 |
| L1 | 无头浏览器（Headless Chrome） | JS渲染页面 |
| L2 | 有头浏览器+DOM操作 | 登录、填表、点击 |
| L3 | 截图+视觉识别 | 图片中的文字/图表 |

### 远程接管：KasmVNC

- 通过网页远程连接服务器桌面，处理验证码等AI无法解决的场景

## 三、文件同步方案

| 方案 | 优劣 |
|------|------|
| 坚果云 | 简单直观，但成本高、有流量限制 |
| rclone远程挂载（推荐） | 免费、实时读写、Mac原生支持 |
| File Browser（手机端） | 浏览器访问服务器文件，支持阅读编辑 |

rclone 核心命令：

```bash
brew install rclone
rclone config
rclone nfsmount my-server:/ ~/mnt/my-server --vfs-cache-mode full --sftp-connections 8 --sftp-idle-timeout 0 --daemon
```

## 四、人格配置与深度定制

### SOUL.md 核心要素

- 关系定位：平等同事关系（非助手/讨好者）
- 行为准则：务实温和、结论先行、有立场、敢拦错
- 沟通风格：简洁直给，允许中文互联网口语
- 禁忌：禁止废话、禁止急于认同

### 入职培训：知识库注入

将个人知识库（如Obsidian笔记）复制到服务器，让 OpenClaw 读取并更新配置文件，实现深度理解用户。

## 五、日常使用技巧

### 核心命令

| 命令 | 功能 |
|------|------|
| `/status` | 查看当前状态 |
| `/stop` | 终止当前任务 |
| `/model <名字>` | 切换大模型 |
| `/compact` | 压缩上下文 |
| `/think off` | 关闭深度思考 |

### 多场景管理

- 建议创建多个群组（工作群/写作群/闲聊群），避免上下文污染
- steer模式：新消息实时注入任务，无需等待当前任务结束

### 踩坑点

- 2G内存会卡死，至少4G
- 关闭自动重启功能，避免改错配置后死循环
- 排错用 Claude Code，手机远程用 HappyCoder
