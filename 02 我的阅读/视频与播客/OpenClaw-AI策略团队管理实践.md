# OpenClaw AI策略团队管理实践

## 项目背景

- 一周内消耗超十亿 token（约 1500 美金）
- 目标：探索 AI 策略团队管理方法，实现 AI 驱动的团队协调

## OpenClaw 核心定位

- 定位：AI agents 团队的协调枢纽，需调教成完美项目经理
- 非开箱即用，需通过 agents.md 配置行为准则（任务边界、汇报机制、协作流程）
- 初始状态类似未培训的新员工，需持续优化

## 三大核心能力增强

### 1. 主动性提升

- 痛点：传统 agents 被动执行，无主动问题解决与复盘能力
- 解法：引入 Heartbeat（心跳机制），定时检查项目卡点并主动汇报
- 配置自动化汇报规则，无需人工询问

### 2. 可重入性优化

- 定义：中断后能从断点稳定恢复并继续推进
- 工作记忆管理：从被动记录改为主动压缩上下文 + 刷新 Session
- 长期任务支持：横跨多天的任务中保持对推进路径的清晰认知

### 3. AI 式思考（Gentle Thinking）

- 核心：AI 应采用自身高效模式解决问题，而非模仿人类行为

| 实现方式 | 效率 | Token 消耗 | 优先级 |
|----------|------|-----------|--------|
| 模拟人类操作浏览器 | 最慢 | 最高 | 最低 |
| 编写 Playwright 脚本 | 中等 | 中等 | 中等 |
| 直接调用平台 API | 最快 | 最低 | 最高 |

- 扩展：agents 间通过共享工作记忆日志路径传递 context，替代低效的"会议聊天"模式

## 我的收获

- agents.md 的配置思路可借鉴到自己的 CLAUDE.md 体系
- 心跳机制的概念值得关注，当前记忆系统的"启动序列"有类似思路
- API 优先于浏览器模拟的原则，与 read-url Skill 的设计方向一致
