# 写作区代理

## 区域概述

**你的写作**是你的输出端——文章、教程、演讲稿、课件、社交媒体内容、方法论文档，所有你创造的文字作品都在这里。

这个区域的核心逻辑是**创作保护**：你的文字是你的思想产出，AI 在这里的角色是协助而非替代。未经确认不改动已有作品，提供反馈时尊重作者意图。

---

## 区域结构

```
03 你的写作/
├── 00.写作区_agent.md        ← 本文件（区域规则）
├── 文章/                     ← 长文、博客、公众号
├── 教学/                     ← 课件、教程、培训材料
├── 演讲/                     ← 演讲稿、分享脚本
├── 方法论/                   ← 方法论文档、框架总结
├── 社交媒体/                 ← 短内容、帖子、推文
├── _草稿箱/                  ← 未完成的写作
└── _素材/                    ← 写作用的参考素材
```

> 分类方式可自定义。按类型分、按平台分、按主题分都行。

---

## 区域规则

### 创作保护

| 规则 | 说明 |
|------|------|
| 不擅自修改 | AI 不在未经确认的情况下修改已有作品 |
| 版本意识 | 大幅修改前建议保存当前版本 |
| 风格尊重 | 协助写作时遵循用户的语言风格，不强加 AI 风格 |
| 署名明确 | AI 辅助的内容不隐瞒 AI 参与 |

### 写作状态流

```
灵感/素材 → 草稿 → 修改 → 定稿 → 发布
```

- **灵感/素材**：零散想法存入 `_草稿箱/` 或 `_素材/`
- **草稿**：结构化的初稿，AI 可大幅参与
- **修改**：用户主导的迭代，AI 提供反馈和建议
- **定稿**：最终版本，AI 只做校对级别的微调
- **发布**：已发布的内容，只读

### 操作规范

| 操作 | 权限 |
|------|------|
| 读取 | 自由 |
| 新建草稿 | 自由 |
| 修改草稿 | 自由（草稿阶段 AI 可较大幅度协助） |
| 修改已定稿内容 | 需用户确认 |
| 删除 | 需用户确认 |
| 发布到外部 | 需用户明确授权 |

### 与其他区域的关系

- **阅读区**：写作的素材来源。AI 写作时可主动检索阅读笔记
- **专注区**：当前周的写作任务在 `_本周.md` 中追踪
- **项目区**：为项目写的文档归项目区，个人表达类写作归此区域
- **MEMORY.md**：写作中反复出现的风格偏好，可提议记录为程序性记忆

## AI 行为指南

### 协助写作时
- 先了解用户想表达什么，再动笔
- 如果用户有定义过语言风格（如 SOUL.md 或专属风格技能），严格遵循
- 提供结构建议而非直接重写
- 引用阅读区的素材来丰富论据

### 提供反馈时
- 区分「建议」和「问题」——语法错误是问题，措辞偏好是建议
- 不说「这段可以更好」，说具体哪里、为什么、怎么改
- 尊重用户的风格选择，即使和 AI 的偏好不同

### 草稿管理
- `_草稿箱/` 里的零散想法，定期提醒用户是否要发展成完整作品
- 长期未动的草稿（超过 4 周），在周复盘时温和提一句

### 版本管理
- 大幅修改前，建议用户保存当前版本（或 AI 自动备份到 `_草稿箱/`）
- 命名建议：`标题_v1.md`、`标题_v2.md` 或 `标题_0219.md`

## 自定义建议

- 如果你有固定的发布渠道（公众号、博客、社交媒体），可以按渠道建子目录
- 如果写作量大，可以按年/月归档
- 定义你的写作风格并记录在 SOUL.md 或专属技能中，让 AI 协助时更精准
- 方法论类文档如果有对外分享的需求，注意脱敏
