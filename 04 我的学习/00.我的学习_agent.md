# 我的学习 · 区域代理

## 定位

主动攻克型学习区。每个课题一个文件夹，通过交互式学习工作流逐步深入，形成自适应学习梯度。

与「02 我的阅读」的区别：阅读是「我看了什么」（被动沉淀），学习是「我要搞懂什么」（主动攻克）。

## 目录结构

```
04 我的学习/
├── 00.我的学习_agent.md
├── [课题名]/
│   ├── 01.md
│   ├── 02.md
│   └── ...
```

## 交互式学习工作流

### 开启新课题

1. 用户说出想学的课题（如「我要搞懂布鲁姆教育理论」）
2. 创建课题文件夹，生成 `01.md` 作为入门篇
3. `01.md` 内容定位：从用户当前认知水平出发，不过于简单也不过于跳跃

### 学习循环

1. 用户阅读当前文章
2. 用户在文章末尾写下：问题、感悟、反馈（任意组合）
3. AI 生成下一篇前，必须先读取用户在前一篇中的所有反馈
4. 基于用户的理解程度和兴趣方向，调整下一篇的内容深度和方向
5. 循环往复，直到用户认为已掌握

### 文章格式

```markdown
---
topic: [课题名]
seq: [序号]
created: [日期]
---

# [课题名] · [序号] [本篇主题]

[正文内容]

---

## 我的反馈

> 在这里写下你的问题、感悟、反馈，AI 会基于此生成下一篇。

```

### 动态调整规则

- 用户反馈显示理解良好 → 加深难度，跳过冗余
- 用户反馈显示困惑 → 降低难度，补充基础，换个角度解释
- 用户表达出对某个分支的兴趣 → 顺着兴趣方向展开
- 用户明确说「这部分我懂了」→ 直接跳到下一个知识点

### 核心原则

- 永远使用中文
- 内容贴合用户的实际认知水平，不假设用户已知
- 每篇聚焦一个核心概念，不贪多
- 鼓励用户写反馈，反馈越具体，下一篇越精准
