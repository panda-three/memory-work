# 技能库区域代理

## 区域定位

**技能（Skill）** 是可执行的能力模块，比静态工具更强。一个技能可以：
- 运行脚本或命令
- 生成输出（日历、报告、分析）
- 触发工作流或流程
- 管理系统操作
- 对接外部服务

技能是知识系统与实际自动化之间的桥梁。

## 目录结构

```
06 我的工具/
├── datetime-check/
│   ├── SKILL.md              ← 技能说明书（必须）
│   ├── scripts/              ← 可执行脚本（可选）
│   ├── references/           ← 参考资料（可选）
│   └── assets/               ← 模板素材（可选）
├── memory-review/
│   ├── SKILL.md
│   └── ...
├── read-url/
│   ├── SKILL.md
│   └── ...
└── 00.skills_agent.md        ← 本文件（区域代理）
```

### 单个技能的文件结构

```
skill-name/
├── SKILL.md               ← 必须，技能说明书
│   ├── YAML frontmatter
│   ├── 用途
│   ├── 触发方式
│   ├── 输入参数
│   ├── 输出格式
│   ├── 依赖项
│   └── 执行指令
├── scripts/               ← 可选，可执行脚本
├── references/            ← 可选，参考资料
└── assets/                ← 可选，模板素材
```

## SKILL.md 格式

### Frontmatter（必填）

```yaml
---
name: skill-name           # kebab-case 命名
type: protocol | skill     # 协议型 或 技能型
trigger: 触发条件描述
version: 1.0
created: YYYY-MM-DD
---
```

### 正文结构

```markdown
# 技能名称

## 用途
这个技能做什么、为什么需要它。

## 触发方式
什么时候运行、谁来触发。

## 输入参数
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| ... | ... | ... | ... |

## 输出格式
返回什么、格式是什么。

## 依赖项
需要什么环境、工具或其他技能。

## 执行指令
AI 读到这里后，按以下步骤执行：
1. ...
2. ...
```

## 两种运行机制

### 协议型（Protocol）—— 条件反射

- 在 CLAUDE.md 启动序列中定义
- **每次会话自动执行**，无需用户触发
- 例：datetime-check、专注区同步
- 类比：后台反射，必须执行，不可跳过

### 技能型（Skill）—— 有意识的动作

- AI 根据用户意图语义匹配触发
- 例："保存对话"、"生成日历"、"做个复盘"
- 用户不需要说出精确的触发词，AI 理解意图即可
- 类比：按需调用，灵活适应

## AI 在此区域的行为规则

- **创建**：写 SKILL.md 之前先向用户提议（说明用途和触发方式）
- **修改**：更新已有技能时做版本管理
- **测试**：新技能注册前先验证可用
- **去重**：创建前检查是否已有类似技能

## 最佳实践

1. **单一职责**：一个技能 = 一个能力
2. **触发清晰**：明确定义何时、如何运行
3. **参数可复用**：接受输入，不硬编码
4. **优雅容错**：失败时给出有用的错误信息
5. **有文档**：每个技能都要有清晰的用途和用法说明
6. **有版本**：追踪变更，保持兼容
7. **有示例**：SKILL.md 中包含使用示例

## 组织规则

- 按字母或功能分组
- 命名统一用 `kebab-case`（如 `datetime-check`）
- 文件结构保持一致
- `scripts/` 与文档分开存放
- 旧版本技能加时间戳归档
