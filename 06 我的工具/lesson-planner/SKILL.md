# lesson-planner：通用备课技能

## 技能说明

这是一个通用的备课技能，专为中职教师设计。能够：
1. 分析教学内容，判断需要几节课
2. 为每节课设计精确的 45 分钟时间表
3. 设计分层练习（难度循序渐进）
4. 生成 PPT 制作指南（用户手动制作 PPT，5-10 分钟）
5. 生成辅助材料（流程卡、反思模板）

**核心特点**：
- 适配基础差的学生（大量练习 + 高频反馈）
- 支持两种输入：PPT 文件 或 主题描述
- 不生成 PPT 文件（成本高且难以匹配用户风格）
- AI 提供所有素材，用户复制粘贴即可

## 模型分工

| 阶段 | 使用模型 | 原因 |
|------|----------|------|
| 容量评估 + 课程设计 | Opus | 需要深度分析和规划能力 |
| PPT 制作指南 + 文档生成 | Opus 或 Sonnet | 文档生成任务，两者均可 |

**说明**：不再生成 PPT 文件，改为生成制作指南，用户手动制作更快更精确

## 执行流程

### 阶段 1：容量评估（使用 Opus）

**输入**：PPT 文件路径 或 主题描述

**步骤**：
1. 读取 `references/00-容量评估.md`，了解评估标准
2. 如果是 PPT：
   - 使用 pptx 技能提取文本内容
   - 识别核心概念
3. 如果是主题：
   - 根据主题拆解知识点
   - 估算每个知识点的难度
4. 执行容量评估：
   - 核心概念数量
   - 认知难度评分（1-5 分）
   - 练习时间估算
   - 依赖关系分析
5. 判断需要几节课
6. 生成 `容量评估报告.md`（保存到工作目录）

**输出**：
- `容量评估报告.md`
- 课程拆分方案

---

### 阶段 2：课程设计（使用 Opus）

**输入**：容量评估结果

**步骤**：
1. 读取 `references/01-内容规划.md`，了解内容规划原则
2. 读取 `references/02-时间分配.md`，了解时间分配公式
3. 读取 `references/03-练习设计.md`，了解练习设计标准
4. 为第 1 课设计详细方案：
   - 教学目标（知识、能力、情感）
   - 精确的 45 分钟时间分配
   - 5 个提问点设计
   - 8-12 个练习设计（分 3 个难度梯度）
   - 作业布置
   - 风险应对
5. 生成 `第N课-[主题]-设计方案.md`

**输出**：
- `第N课-[主题]-设计方案.md`

---

### 阶段 3：生成 PPT 制作指南

**输入**：课程设计方案

**步骤**：
1. 生成 PPT 制作指南：
   - 提问页文案（5 个）
   - 练习页内容（8 个题目 + 8 个答案）
   - 插入位置建议（在原 PPT 的哪一页后插入）
   - 格式建议（颜色、字号、布局）
   - 快速制作步骤（预计 5-10 分钟）
   - 检查清单
2. 生成课堂流程卡：
   - 使用 `assets/流程卡模板.md`
   - 精确到分钟的时间表
   - 可打印的 A4 版本
3. 生成教学反思模板：
   - 使用 `assets/反思模板.md`
   - 课后填写用

**输出**：
- `第N课-PPT制作指南.md`（用户根据此指南手动制作 PPT）
- `第N课-课堂流程卡.md`
- `第N课-教学反思模板.md`

**为什么不生成 PPT**：
- PPT 生成成本高（Token + 时间）
- AI 生成的 PPT 难以完美匹配用户风格
- 用户手动制作 PPT 更快更精确（5-10 分钟）
- AI 提供所有素材，用户只需复制粘贴

## 关键约定

### 学生特点
- 中职学生，基础较弱
- 需要大量练习才能掌握
- 注意力集中时间短（5-8 分钟）

### 时间分配原则
- 讲解：40-50%
- 练习：30-40%
- 提问：10-15%
- 总结：2-5%

### 练习设计原则
- 难度 1（基础）：6-8 个，每个 2-3 分钟
- 难度 2（组合）：2-4 个，每个 3-4 分钟
- 难度 3（综合）：1-2 个，作为作业

### 互动频率
- 每 5-8 分钟必须有互动（提问或练习）
- 提问点：5 个左右
- 练习点：8-12 个

## 使用示例

### 示例 1：基于 PPT 备课
```
用户：用 lesson-planner 分析这个 PPT：01 我的项目/教学相关/中职/电子技术/21-基本逻辑门电路.pptx

AI：
1. 读取 references/00-容量评估.md
2. 使用 pptx 技能提取 PPT 内容
3. 执行容量评估
4. 生成容量评估报告
5. 询问用户是否继续设计第 1 课
6. 用户确认后，生成详细设计方案
7. 生成 PPT 制作指南（用户根据指南手动添加互动页）
8. 生成流程卡和反思模板
```

### 示例 2：基于主题备课
```
用户：用 lesson-planner 设计一节课：Python 列表操作

AI：
1. 读取 references/00-容量评估.md
2. 拆解知识点：列表创建、索引、切片、增删改查
3. 执行容量评估
4. 生成容量评估报告
5. 询问用户是否继续设计第 1 课
6. 用户确认后，生成详细设计方案
7. 生成 PPT 制作指南（提供所有提问和练习的文案）
8. 生成流程卡和反思模板
```
