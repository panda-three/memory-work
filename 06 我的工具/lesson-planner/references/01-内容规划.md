# 内容规划模块

## 目标

为单节课规划教学内容，确保知识点清晰、逻辑连贯、目标明确。

---

## 教学目标设计

### 三维目标框架

**1. 知识目标**（Know）
- 学生需要记住什么？
- 学生需要理解什么？
- 学生需要掌握什么？

**2. 能力目标**（Do）
- 学生能够做什么？
- 学生能够应用什么？
- 学生能够分析什么？

**3. 情感目标**（Feel）
- 学生的学习态度
- 学生的学习信心
- 学生的学习兴趣

### 目标编写原则

**SMART 原则**：
- **Specific**（具体）：不要写"理解逻辑门"，要写"掌握与门的真值表"
- **Measurable**（可测量）：能够通过练习验证是否达成
- **Achievable**（可达成）：符合学生当前水平
- **Relevant**（相关）：与课程主题直接相关
- **Time-bound**（有时限）：45 分钟内能够达成

### 目标数量建议

- 知识目标：3-5 个
- 能力目标：2-3 个
- 情感目标：1-2 个

---

## 内容组织原则

### 1. 从具体到抽象

**错误示例**：
```
1. 讲解逻辑代数
2. 讲解真值表
3. 举例：与门
```

**正确示例**：
```
1. 举例：两个开关控制一个灯（具体）
2. 抽象：与逻辑关系
3. 符号化：与门、真值表
```

---

### 2. 从简单到复杂

**难度梯度**：
```
单一知识点 → 两个知识点组合 → 多个知识点综合
```

**示例（与门教学）**：
```
1. 与门的定义（简单）
2. 与门的真值表（中等）
3. 3 输入与门（复杂）
4. 与门的实际应用（综合）
```

---

### 3. 从已知到未知

**连接策略**：
- 从学生的生活经验出发
- 从学生已学的知识出发
- 用类比和对比建立联系

**示例**：
```
已知：开关的开/关
未知：逻辑状态的 0/1
连接：开关打开 = 1，开关关闭 = 0
```

---

## 内容结构设计

### 标准结构（45 分钟）

```
导入（3-5 分钟）
    ↓
前置知识复习（5-8 分钟）
    ↓
新知识讲解（15-20 分钟）
    ├─ 知识点 1（5-7 分钟）
    ├─ 知识点 2（5-7 分钟）
    └─ 知识点 3（5-7 分钟）
    ↓
练习巩固（15-20 分钟）
    ├─ 基础练习（8-10 分钟）
    ├─ 组合练习（5-7 分钟）
    └─ 综合练习（2-3 分钟）
    ↓
总结（2-3 分钟）
```

---

### 导入设计

**导入目的**：
1. 激发兴趣
2. 建立联系
3. 明确目标

**导入方式**：
- **情境导入**：生活中的例子
- **问题导入**：抛出一个问题
- **复习导入**：回顾上节课内容
- **演示导入**：实物演示或视频

**时间控制**：3-5 分钟（不要超过 5 分钟）

---

### 讲解设计

**讲解原则**：
1. **小块化**：每个知识点 5-7 分钟
2. **可视化**：用图表、动画辅助
3. **互动化**：每讲 5-8 分钟必须有互动

**讲解结构**：
```
是什么（定义）
    ↓
为什么（原理）
    ↓
怎么用（应用）
```

**示例（与门讲解）**：
```
1. 是什么：与门是实现"与"逻辑的电路
2. 为什么：当所有输入都为 1 时，输出才为 1
3. 怎么用：两个开关控制一个灯
```

---

### 练习设计

**练习分层**：
- **难度 1（基础）**：单一知识点，机械练习
- **难度 2（组合）**：两个知识点组合
- **难度 3（综合）**：多个知识点综合应用

**练习数量**：
- 难度 1：6-8 个（每个 2-3 分钟）
- 难度 2：2-4 个（每个 3-4 分钟）
- 难度 3：1-2 个（作为作业）

**练习类型**：
- 填空题：适合记忆性知识
- 判断题：适合概念辨析
- 选择题：适合规律理解
- 计算题：适合公式应用
- 设计题：适合综合应用

---

### 总结设计

**总结内容**：
1. 本节课的核心知识点（3-5 个）
2. 关键口诀或公式
3. 作业布置

**总结方式**：
- **教师总结**：快速回顾
- **学生总结**：请学生说出收获
- **对比总结**：对比本节课与上节课的联系

**时间控制**：2-3 分钟

---

## 内容规划检查清单

### 必检项

- [ ] 教学目标是否明确？
- [ ] 知识点是否按难度梯度排列？
- [ ] 每个知识点是否有对应的练习？
- [ ] 是否每 5-8 分钟有互动？
- [ ] 总时间是否控制在 45 分钟内？

### 优化项

- [ ] 是否有生活化的例子？
- [ ] 是否有可视化的图表？
- [ ] 是否有对比和类比？
- [ ] 是否有学生容易犯的错误提示？
- [ ] 是否有拓展内容（选做）？

## 常见问题

### Q1：一节课讲几个知识点合适？

**A**：
- 基础差的学生：1-2 个核心知识点
- 基础中等的学生：2-3 个核心知识点
- 基础好的学生：3-4 个核心知识点

**判断标准**：每个知识点需要 15-20 分钟（讲解 + 练习）

---

### Q2：如何判断知识点是否讲透了？

**A**：
- 学生能够用自己的话解释
- 学生能够举出新的例子
- 学生能够完成相关练习（正确率 ≥ 80%）

---

### Q3：如何处理超时？

**A**：
- **预防**：设计时留 5 分钟弹性时间
- **应对**：压缩总结时间，或将部分练习作为作业
