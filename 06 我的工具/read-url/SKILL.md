# read-url

## 用途

读取用户提供的 URL 内容。当直接抓取失败时（如微信公众号防爬），自动通过 Jina Reader 中转提取正文。

## 触发方式

技能型——用户提供 URL 并表达阅读/提取/总结意图时触发。语义匹配示例：

- "帮我读一下这篇文章"
- "这个链接讲了什么"
- "总结一下这个 URL"
- "提取这篇文章的内容"

## 执行流程

1. 先直接抓取用户提供的原始 URL
2. 如果失败（防爬、验证墙、内容为空），自动在 URL 前拼接 Jina Reader 前缀重试：
   ```
   https://r.jina.ai/<原始URL>
   ```
3. 提取到正文后，根据用户意图处理（总结、存档、分类等）

## 输入

- `url`（必填）：目标网页地址
- `intent`（可选）：用户希望对内容做什么（默认：提取正文）

## 输出

- 网页正文的 Markdown 格式内容
- 如用户有后续意图（总结、存档），继续执行对应操作

## 注意事项

- Jina Reader 免费，无需 API Key
- 不保证 100% 成功，部分页面可能仍无法提取
- 如果 Jina 也失败，提示用户手动复制正文
