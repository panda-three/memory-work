# 封闭平台内容抓取方案

## 问题

公众号、抖音、视频号等封闭平台的分享链接，直接抓取拿不到内容（JS 动态渲染 + 防爬）。

## 解决方案：Get笔记 + Jina Reader

核心思路：用 Get笔记 app 作为中间层，把封闭平台内容转成可抓取的链接。

### 流程

```
手机上看到好内容
  → 复制分享链接（公众号/抖音/视频号/B站均可）
  → 粘贴到 Get笔记，自动提取内容
  → Get笔记生成自己的分享链接（d.biji.com/xxx）
  → 把 Get笔记链接发到收件箱
  → AI 启动时用 read-url Skill 抓取：
      curl -s "https://r.jina.ai/<Get笔记链接>"
  → 整理成结构化笔记入库
```

### 为什么有效

- Get笔记在手机端完成内容提取（绕过了平台封闭性）
- Get笔记的分享链接是标准 Web 页面，Jina Reader 能正常解析
- Jina Reader 返回 Markdown 格式，直接可用

### 适用平台

| 平台 | 内容类型 | 验证状态 |
|------|----------|----------|
| 微信公众号 | 文章 | 待验证 |
| 抖音 | 视频笔记 | 待验证 |
| 视频号 | 视频笔记 | 待验证 |
| B站 | 视频笔记 | 待验证 |
| 笔记类 app | 图文笔记 | ✅ 已验证（biji.com） |

### 局限性

- 图片：Jina Reader 提取的是文本，图片信息会丢失。关键图表需手动截图补充
- 依赖 Get笔记 app 的内容提取质量
- 需要手机端多一步操作（复制链接到 Get笔记）

## 历史方案（已废弃）

之前尝试过 Playwright 本地浏览器方案抓取公众号，存在以下问题：
- chromium 体积大（~150MB），依赖重
- 微信防爬机制复杂，不确定能绕过
- 方案偏重，维护成本高

Get笔记方案更轻量、更通用，覆盖多个平台，不需要本地安装额外依赖。
