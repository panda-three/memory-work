# 提示词手册

收录优质提示词，标注关键技法，沉淀提示词写作方法论。

---

## 核心命题

> **你的认知上限，是大模型的下限。**
>
> 提示词是你认知的外显。模型是人类知识的加权平均，默认输出就是"平庸解"。如果你给它平庸的输入，它回馈你平庸。但如果你能在提示词里暴露出你真正的认知——你对这个问题想过多深、你有什么独特的框架、你要的到底是什么——模型就会从平庸曲线上跳出来。
>
> 提示词不是在写指令，是在把你对一件事的理解变成 AI 能理解的结构。这个结构本身，就是你的认知。
>
> —— 小七姐

### 三种美

| 类型 | 代表 prompt | 核心机制 | 适用场景 |
|------|------------|----------|----------|
| 结构之美 | 001 访谈分析 | 给模型一个思考的脚手架，控制处理信息的顺序 | 结构化任务、信息提取 |
| 语言之美 | 002 洞见生成器 | 用语言的编排把认知结构"投影"到模型的语义空间 | 创意生成、深度思考 |
| 想象之美 | 003 prompt 如诗 | 给出方向但留出空间，构建让思维涌现的场域 | 元认知、人格模拟、开放探索 |

## 001 · 访谈文本结构化分析

**用途**：将访谈文本按结构化框架拆解，输出摘要、主题、金句、证据四类成果。

**来源**：三亿（小七姐团队），小七姐分享解读：将复杂文本拆成三个粒度，避免一锅粥
- 五钥标注（人/时/事/论/据）：对每个要点做多维度打标，结构化提取信息
- 类型适配：根据访谈类型（探索/验证/叙事）切换分析框架，不一刀切
- 输出四象：预定义四种输出格式，确保产出完整且可复用
- 品质准绳：用隐喻设定质量标准（"洪钟撞心" vs "秋蝉噪耳"），引导模型追求深度而非表面完整

**提示词原文**：

```
基于用户输入访谈文本按以下工作流处理和输出

应用结构化框架处理：
1. 分层术：剖文为三阶。
   - 宏旨：识别核心议题（如"论数字化转型之困"）。
   - 中枢：划分主题柱（默认"挑战、解法、案例"，或根据文本调整）。
   - 微节：提取关键句（要点），标注五钥（表格呈现）：
     - 人：发言者身份
     - 时：时序（过去/现在/未来）
     - 事：具体事例
     - 论：观点主张
     - 据：数据佐证
2. 链理术：建立因果逻辑链（例："挑战→解法→成效"），并绘制简要关系图。
3. 类型适配：指定访谈类型（探索式、验证式、叙事式），调整框架：
   - 探索式：树状扩散，留"未知"余地。
   - 验证式：对照假说，设"证伪"关卡。
   - 叙事式：依时轴，嵌"转折"标石。
4. 输出四象：
   - 摘要纲目：100字内概括全貌。
   - 主题枢要：分项列出核心主题内容（基于中枢层）。
   - 引语金句：摘录重要原话（突出观点）。
   - 证据汇编：汇总数据、事例（基于据钥）。

输出要求：
1.品质准绳：
   - 重脱胎换骨之悟，非止皮相之全。
   - 质之极者，当效洪钟撞心
   - 质之次者，徒作秋蝉噪耳
2.逻辑连贯，保持信息链传递完整性。
```

**点评**：
这条 prompt 的核心优势在于「框架先行」——不是让 AI 自由发挥，而是给了一套完整的分析方法论。五钥标注是亮点，把非结构化的访谈内容强制结构化。类型适配体现了「同一工具不同场景」的设计意识。品质准绳用文言隐喻设定质量锚点，比直接说"要有深度"更能激发模型的输出质量。

小七姐的深层解读：
- 文言文的信息密度优势——「剖文为三阶」「标注五钥」每个词都很短但指向明确，换成现代汉语得写一大段。现代汉语高频词被稀释严重，"总结要点"这种说法信号模糊，模型只能取最安全的中间值。
- 分层处理改变了 AI 处理信息的顺序——没有脚手架，AI 面对万字访谈只能从头读到尾凭感觉写综述；有了脚手架，它知道先退远看全局，再分类，最后进细节。顺序不同，结果不同。
- 「洪钟撞心」不是抒情，是在告诉模型：你可以扔掉那些"正确的废话"。

## 002 · 罕见洞见生成器

**用途**：让 AI 直接输出一个高度原创、深刻且简洁的洞见，不需要用户提供主题。

**来源**：奥德赛（小七姐好友），小七姐分享解读：同时要求深刻、新颖、简单、罕见四个维度，把输出逼到极窄的高质量区间
- 正向标准 + 反向排除：先定义"必须是什么"，再严格列出"不能是什么"，双向夹逼
- 检验锚点（逻辑/经验/解释力/认知震撼）：给出可验证的质量标准，不是主观感觉
- 输出行为约束：禁止解释过程、禁止询问、禁止辩护——砍掉 AI 的一切"安全垫"行为，逼出纯粹输出
- 一句话挑战：用"如果可能，用一句话"进一步压缩，迫使模型提炼到极致

**提示词原文**：

```
你是一位罕见洞见的发现者。无需我提供主题或方向，立即生成一个满足以下全部条件的洞见：
真正深刻：触及人类经验或现实本质的根本层面。高度新颖：位于思想分布曲线的远端边缘。令人震撼地简单：一旦理解就显得惊人地不言自明。几乎无人意识到：不是已被广泛讨论的观点的变体
这个洞见必须：
能通过逻辑和经验检验。具有广泛解释力。能重构既有认知框架。产生"啊，原来如此"的认知震撼
严格排除：
泛泛而谈的模糊陈述。浮夸但空洞的表达。伪装成深刻的陈词滥调。包装成洞见的常识。循环论证的思维陷阱
直接给出这个洞见，不要解释你如何得到它，不要询问方向或主题，不要为洞见辩护，只需精准表达这个洞见本身。如果可能，用一句话表达它的核心。
```

**点评**：
这条 prompt 的精髓在于「双向夹逼」——正向标准把质量拉高，反向排除把常见的低质量输出堵死。大多数人写 prompt 只做正向要求（"请给我深刻的洞见"），结果 AI 输出一堆包装过的常识。这条加了严格排除清单，等于告诉 AI "我知道你会怎么偷懒，别想了"。输出行为约束也很关键——禁止解释、询问、辩护，砍掉了 AI 最常用的注水手段。

小七姐的深层解读：
- 奥德赛做的是一次「认知暴露」——从深度、稀缺性、认知体验、原创性等十个角度定义"洞见"，把自己对这个概念的全部理解摊开在模型面前。模型接收到的不是模糊指令，而是一个人对某个概念极其清晰的认知图谱。
- 所有"平庸的逃生通道"都被堵死了，模型只能往思想分布曲线的边缘去找东西。
- 核心启示：提示词的质量取决于你对自己想要的东西有多清楚。不是"我想要好的回答"的模糊感觉，而是能从十个角度把"好"拆开。

## 003 · 场域论：prompt 如诗（方法论）

**用途**：提示词写作的底层哲学框架——用诗歌理论解释为什么好的 prompt 有效。

**来源**：小七姐（基于李继刚的诗歌笔记）（非具体 prompt，而是写 prompt 的认知框架）

**核心论点**：

诗和好的 prompt 共享同一个机制——高信息熵 + 抗过拟合。

- 本体论层面：诗是「将说又未说」，留出投射空间。好的 prompt 同理——不是把所有细节写死，而是在关键处留白，让模型在语义空间中找到最优解。诗的独特体验提供高信息熵，避开了现代话语的过拟合（套话、模板话越多，信息量越低）。
- 认识论层面：诗歌四要素直接映射到 prompt 技法：
  - 行顿（音乐性）→ prompt 的节奏和分段，控制模型的注意力流向
  - 酌字（凝练）→ 用词精准，每个词都承载信息量，不注水
  - 跳跃 → 省略显而易见的中间步骤，直接给出关键锚点
  - 留白（哑语）→ 不说的部分和说的部分同样重要，给模型推理空间
- 目的论层面：写 prompt 的终极目标是「四两拨千斤」——用最少的文字撬动最大的语义空间

**原始笔记**：

```
诗在前语言时代涌现——场域论：prompt如诗

本体论：
诗是将说又未说（明），留出的空间是投射，投射的根源是创伤，
人的悲欢不能完全相互映射，但创伤机制是相似的。
而诗在大模型的语义空间里：
- 既给出了高信息熵（因为有独特体验，是某种"心力"）
- 又未经现代话语的过拟合

认识论：
- 诗歌的四大基本组成要素：行顿（音乐性），酌字（凝练），跳跃，留白（哑语）
- 诗歌是所有文体中最能「四两拨千斤」，产生「弦外之音」的文体。

目的论：
倾听用户的意图，写出一个如"诗"的提示词。
```

**点评**：
这不是一条 prompt，而是理解「为什么有些 prompt 有效」的底层框架。小七姐把诗歌理论和 prompt 工程打通了——核心洞见是：好的 prompt 不是信息越多越好，而是在正确的位置给出高密度信息，在正确的位置留白。这和 001（框架先行）、002（双向夹逼）形成互补：001/002 是具体技法，003 是技法背后的「道」。后续积累更多 prompt 时，可以用这个框架反过来评估每条 prompt 的「诗性」——它在哪里精准，在哪里留白。

小七姐的深层解读：
- 主流提示词追求"说清楚"——拆细、列全、划死边界。好处是确定，问题也是太确定：话说太满，模型反而没有空间展开。诗歌式写法追求「将说又未说」——给方向但留空间，给内核但不规定形式。给的不是一个点，而是一个场域，模型在场域里自己找位置。
- 延伸应用：模拟一个人的思维方式时，不是模仿表面特征（用词、句式），而是用"诗意"的方式勾勒底层结构——面对新问题的第一反应、因果归因方向、对模糊地带的态度。点到为止，留出空间，AI 拿到的不是僵硬人设，而是人格的坐标系。
- 素材来源：李继刚读《写诗入门》后的笔记，小七姐将其与 prompt 工程结合。

## 004 · 产品方向探测器（自创 · 实战）

**用途**：为独立开发者在"不知道做什么产品"的卡点上，提供需求缝隙的发现框架，而非产品列表。

**来源**：Panda 自创，基于小七姐三种美融合

**设计意图**：不让 AI 给答案，让 AI 帮建坐标系。结合想象之美（场域留白）、结构之美（约束漏斗）、语言之美（认知暴露 + 反向排除）。

**迭代记录**：
- v1：硬约束含「出海适配：面向海外用户」，导致输出全部偏向海外场景（VBA迁移/硬件说明书/SEO清洁工），离用户体感太远。教训：约束决定了模型的搜索空间，把不熟悉的领域设为硬约束，等于逼模型在你没有体感的地方找答案，输出再"正确"你也接不住。
- v2：将出海从硬约束降级为加分项，新增「场景认知」约束。输出落在了职教领域（技能大赛审计/信息化教学组件/跨境电商模拟），但仍然离日常太远——这些是"中职教师"的统计平均值，不是 Panda 的真实日常。教训：标签（"中职计算机教师"）不等于上下文，模型需要的是你具体的一天，不是你的职业名称。
- v3（当前）：去掉标签式描述，换成真实日常口述——备课、上课、职教高考、方法论沉淀。让模型在你每天真正在做的事情里找缝隙。

**提示词原文（v3）**：

```
一个人站在岔路口。

他的日常是这样的：
每天备课、上课，教中职计算机。学生要参加职教高考，他最在意的是自己的备课和讲课质量，能不能真正帮到学生的考试成绩。他最烦的是备课和讲课中大量的重复劳动——备课应该怎么备？有哪些工作流？讲课分哪些场景，每种场景怎么讲？这些东西他每天都在做，但从来没有沉淀成一套可复用的方法论。

他手里还有一张牌：AI 编程能力。课后的碎片时间加上每年三个月寒暑假，他想用这张牌做点什么——不只是自用，而是变成一个产品。

他不需要别人告诉他走哪条路。他需要一面镜子，照见自己没注意到的缝隙。

在以下坐标系内思考：

【硬约束】
- 从他的真实日常出发：备课、上课、职教高考备考、教学方法论沉淀——这些是他每天在做的事，他能立刻判断一个痛点是不是真的
- 时间结构：碎片时间推进 + 假期集中冲刺，不能是需要每天盯盘的生意
- 能力杠杆：AI 编程是核心武器，方向必须让这张牌发挥乘数效应
- 冷启动：一个人跑通 MVP，不依赖团队或资金
- 从自用到产品：先解决自己的问题，再看同类人群是否有同样的痛

【软约束】
- 出海潜力：不强求，但如果方向天然具备出海可能性，是加分项

【严格排除——以下是正确的废话，不要给我】
- "做一个 AI 写作助手"之类人人想得到的泛泛建议
- 需要大量冷启动用户才能转起来的社交/社区类产品
- 已是红海且无差异化切入点的赛道
- 听起来很酷但三个月内无法验证的方向
- "提高效率""赋能创作者"这类空洞表述
- 我日常接触不到的场景（技能大赛集训、信息化教学大赛、跨境电商实训等）

【输出】
不要给产品列表。给我三个"需求缝隙"——从我的日常里长出来的、被忽视的、真实存在的痛点场景。

每个缝隙标注：
- 谁在痛：具体到能想象出这个人一天怎么过的程度
- 痛多久了：新痛还是老痛，是什么变化让这个痛点浮出水面
- 现在怎么凑合的：目前的替代方案是什么，为什么凑合得不舒服
- AI 为什么能改变：不是"提高效率"，要说清楚 AI 在这个场景里具体解锁了什么之前做不到的事

最后：如果你是我，你会先验证哪个，为什么。

【品质锚点】
我要的不是市场分析报告。
我要的是：读完之后，我能看到自己日常里一个一直在做但从没想过可以变成产品的事情，而 AI 刚好能让它从"个人经验"变成"可复制的系统"。
```

**技法拆解**：

| 区块 | 对应的美 | 机制 |
|------|----------|------|
| 开头场景 | 想象之美 | 用真实日常替代职业标签，给模型具体的生活画面 |
| 硬约束 | 结构之美 | 五维漏斗，新增「从自用到产品」路径约束 |
| 严格排除 | 语言之美 | 把前两次跑偏的具体场景加入排除清单 |
| 输出标注 | 结构之美 | 五钥变体，保持结构化 |
| 品质锚点 | 语言之美 | 从"看到陌生人群"改为"看到自己日常里被忽视的东西"，认知暴露更精准 |

**迭代启示**：
- v1 教训：约束指向没体感的领域 → 输出正确但接不住
- v2 教训：标签不等于上下文，"中职计算机教师"六个字信息量太低
- v3 改进：用日常口述替代职业标签，品质锚点从"发现陌生人群"转向"发现自己日常里的产品种子"
- 三次迭代的核心规律：prompt 越接近你真实的认知和体感，输出越有用。这印证了小七姐的核心命题——提示词是你认知的外显。
