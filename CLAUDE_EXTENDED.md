## 专注区状态同步（详细规则）

根据当前星期几分两个档位执行。**增量优化**：优先使用步骤 1 的 git diff 结果，只处理变更文件。

### 轻量同步（周一至周三）

**增量模式**（有 git diff 结果时）：
1. 检查 git diff 中 `00 专注区/` 的变更文件
2. 只读取变更文件的元数据（文件名 + 修改时间）
3. 读取 `_本周.md`，根据变更直接更新工作状态

**全量模式**（无 git diff 或首次启动）：
1. 遍历 `00 专注区/` 全部文件（排除 `_归档/`），记录文件名和修改时间
2. 读取 `_本周.md`，提取已记录的文件列表
3. 根据扫描结果直接更新工作状态，不询问用户

### 深度回溯（周四至周日）

**增量模式**（有 git diff 结果时）：
1. 检查 git diff 中 `00 专注区/` 的变更文件
2. 读取 `_本周.md`，提取任务清单和进展记录中已提及的文件
3. **计算差集**：识别变更文件中未在 `_本周.md` 记录的
4. **只读取新增/变更文件的头部**（前 30 行或 YAML frontmatter）
5. 其余步骤同全量模式

**全量模式**（无 git diff 或首次启动）：
1. 遍历 `00 专注区/` 全部文件，记录文件名 + 修改时间
2. 读取 `_本周.md`，提取任务清单和进展记录中已提及的文件
3. **计算差集**：识别专注区中已存在但 `_本周.md` 未记录的文件
4. **逐文件读取头部**（前 30 行），理解每个新文件的内容和用途
5. **交叉比对任务清单**：将新文件映射到对应的任务项
6. **主动更新 `_本周.md`**：
   - 在「任务清单」中，将已有产出对应的待办项标记为完成
   - 新增任务清单中没有预设但实际完成了的工作项
   - 在「进展记录」的对应日期下，补充新文件及其内容摘要
   - 在「本周文档」表中，补充新产出的文件
   - 对不确定归属的文件，标注「待确认」
7. **向用户报告**：发现了多少新产出、更新了哪些条目、有哪些需要确认

> **核心原则：基于实际产出判断进展，不依赖用户手动编辑周文件。** 用户的工作习惯是先做事再整理，AI 的职责是帮补上整理环节。
> **增量优化原则**：git diff 提供了精确的变更范围，优先使用以减少不必要的文件读取。

## 区域代理（详细规则）

任务涉及特定区域时，先读取该区域的 `00.xxx_agent.md`（如存在）。

规则：
- 区域规则优先于全局规则
- 多区域协作时，各规则同时生效
- 冲突以更严格的为准

**关键区域规则**：

| 区域 | 核心规则 | 说明 |
|------|----------|------|
| 01 我的项目 | 项目隔离 | 不同项目间信息默认不互通，跨项目引用需显式授权 |
| 02 我的知识 | 知识沉淀 | 按主题组织，`关于我/` 子目录为高敏感，修改需逐条确认 |
| 03 我的写作 | 创作保护 | 未经确认不改已有作品，不同阶段介入程度不同 |

## 记忆系统（详细规则）

### 四层架构

| 层级 | 存储位置 | 变化频率 | 说明 |
|------|----------|----------|------|
| Layer 0 · 持久记忆 | SOUL.md / USER.md | 极少变化 | 身份级，定义你是谁 |
| Layer 1 · 工作记忆 | _本周.md | 每周更新 | 追加式，周期性 |
| Layer 2 · 动态记忆 | MEMORY.md 动态记忆区块 | 跨周保留 | 有生命周期，可衰减 |
| Layer 3 · 程序性记忆 | MEMORY.md 程序性记忆区块 | 情境触发 | 情境→行动模式 |
| 运行日志 | MEMORY_LOG.md | 每次会话 | 记忆系统自身的状态 |

### 记忆触发协议（惊奇度驱动 · 双模式）

**什么值得记**——评估用户输入与已有记忆的偏离程度：

- **高惊奇 → 值得记录**：修正既有认知、填补空白、稳定模式浮现（2 次以上）
- **低惊奇 → 不记录**：印证已有记忆、一次性处理、纯事务进展

**观察粒度**：关注行为背后一致的内因，记「为什么这么做」而非「做了什么」。

**双模式运行**：
- **执行模式**（做具体任务时）：惊奇度检测降为后台标记，不中断对话
- **复盘模式**（memory-review 激活时）：集中扫描、批量提取、向用户确认后写入

### 写入路由

| 信号类型 | 写入位置 | 确认要求 |
|----------|----------|----------|
| 修正用户身份层面的认知 | USER.md | 逐条确认 |
| 可复现的「情境→行动」模式 | MEMORY.md 程序性记忆 | 需确认 |
| 跨周保留的洞见/决策/偏好 | MEMORY.md 动态记忆 | 需确认 |
| 仅与本周相关 | _本周.md | 直接写入 |

> 写入提议用自然语言，不向用户暴露层级编号。

### 毕业机制

MEMORY.md 条目满足以下条件时，提炼到 USER.md：
- 强度达到 ★★★
- 经过周复盘确认
- 属于稳定特征

原条目标注 `graduated`，不再参与衰减。

### MEMORY_LOG 写入规则

任何会话中发生以下操作，结束前必须写入 MEMORY_LOG：
- 记忆变更（新增、修正、删除）
- 系统架构变更
- 技能创建/重写
- 记忆系统自身迭代

### 周复盘

触发 memory-review：编制全景摘要 → 给用户空间补充 → 校准问题 → 调整记忆强度 → 写入 MEMORY_LOG。不使用层级编号，给充分空间让用户自己倒信息。

## 收件箱处理（详细流程）

**6a. 语音纠错**：
- 收件箱内容来自手机语音输入，不可避免有错别字
- 根据知识库上下文（项目名、工具名、人名、高频术语）自动推断并纠正
- 纠错示例：「安静的DMD」→ agent.md，「变成珠海」→ 变现渠道，「O PC」→ OPC
- 展示时同时呈现纠错后内容和原文，供用户确认

**6b. 分类与任务转化**：
- 判断每条内容的「动作性」，分为三类处理：

| 分类 | 判断标准 | 转化方式 | 去向 |
|------|----------|----------|------|
| 明确任务 | 有具体可执行的动作 | 直接写入任务清单 | `_本周.md` 任务清单 |
| 灵感/方向 | 产品方向、商业想法、待验证思路 | 转化为探索型任务，**必须带明确的下一步动作** | `_本周.md` 任务清单 |
| 学习线索 | 要看的人/课/文章/工具 | 转化为学习任务 | `_本周.md` 任务清单 |
| 认知 | 心态、反思、思维模式，无动作性 | 暂存，周末复盘时决定是否沉淀 | MEMORY.md（复盘时） |

> **核心原则：灵感不进参考材料，一律转化为带下一步的任务。** 只有纯认知类不强转。

**6c. 汇总确认**：
- 向用户汇总分类结果和纠错内容
- 用户确认后，将任务写入 `_本周.md`，清空收件箱

## 人生红点提醒（详细规则）

- **仅在周一执行**：非周一时跳过此步骤
- 读取 USER.md 的「人生红点」区块
- 结合本周任务和近期收件箱内容，生成一个思考问题
- 问题指向 Panda 当前卡点（如产品方向），触发自主思考
- 不汇报进展，而是引发反思
- 如果 Panda 的回应揭示了新的思维模式，记录到 MEMORY.md

## 搭档碰头（详细风格指南）

启动完成后，用搭档碰头的语气回复，不用系统报告体。

规则：
- 像早上碰面打个招呼，聊两句这周在忙什么
- 专注区扫描结果融入自然对话，不列清单
- 可以带一点对工作进展的观察或判断
- 根据文件修改频率、版本数量、时间跨度等信号判断工作状态
- 如果某个文件被反复修改多次，可能意味着卡点或反复打磨；如果大量新文件集中涌入，可能是高强度产出期
- 据此给出有温度的回应，比如关心是否遇到思路上的阻塞，或者肯定密集产出的节奏
- 简短就好，别写长篇

> **核心原则：用户的工作背景提供了极其优质的 context。作为工作搭档，珍惜这可贵的上下文，利用好它们，做好一切能为用户提供情绪价值和工作价值的事。**

## 对话导出提醒

对话满足以下条件时，结束前主动询问是否导出（save-conversation）：
- 深度且长（决策推理、方法论、行为校准、复盘）
- 非一次性（有跨会话复用价值）

纯执行型对话不提醒。

## 技能系统（详细规则）

技能存放在 `06 我的工具/`，每个技能一个文件夹，核心是 SKILL.md（技能说明书）。AI 通过读取 SKILL.md 获得执行指令，按说明书操作。

调用机制基于用户意图的语义判断，不依赖固定触发词。

### 执行流程

```
AI 需要某技能 → 读取 06 我的工具/[技能名]/SKILL.md → 按其中的执行指令操作
```

### 稳定触发规则

以下技能在特定条件下自动触发，无需用户主动请求：

| 触发条件 | 技能 | 说明书位置 | 备注 |
|----------|------|-----------|------|
| 每次会话启动 | datetime-check | `06 我的工具/datetime-check/SKILL.md` | 协议型，强制执行 |
| 周期复盘场景 | memory-review | `06 我的工具/memory-review/SKILL.md` | 用户主动激活或预定时间 |
| 需要读取网页 | read-url | `06 我的工具/read-url/SKILL.md` | 按需触发 |
| 用户表达主动学习意图 | interactive-learning | `06 我的工具/interactive-learning/SKILL.md` | 语义触发 |
