# Contributing to Memory Work

Thank you for your interest in contributing to Memory Work! This document provides guidelines for contributing to the project.

---

## Ways to Contribute

### 1. Share Your Skills

Have you created a useful skill? Share it with the community!

1. Fork the repo
2. Add your skill to `06 Skills/your-skill/`
3. Include a clear `SKILL.md` with:
   - Purpose and use cases
   - Trigger words
   - Execution instructions
   - Example outputs
4. Submit a pull request

### 2. Improve Documentation

Help make Memory Work more accessible:
- Fix typos or unclear explanations
- Add examples and use cases
- Translate documentation
- Create tutorials or guides

### 3. Enhance Core System

Improvements to the core architecture:
- Memory protocols
- Agent behaviors
- File structure
- Performance optimizations

### 4. Report Issues

Found a bug or have a feature request?
- Open a GitHub issue
- Describe the problem or desired feature
- Include relevant context (your setup, what you tried, etc.)

---

## Development Guidelines

### File Structure

- Keep files in their designated zones
- Follow existing naming conventions
- Use clear, descriptive filenames

### Documentation

- Update relevant documentation when changing functionality
- Include examples for new features
- Keep README.md in sync with changes

### Code Style

- Use clear, descriptive language in prompts
- Follow Markdown best practices
- Keep YAML frontmatter consistent

---

## Submitting Changes

### Pull Request Process

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature-name`)
3. Make your changes
4. Test thoroughly
5. Commit with clear messages
6. Push to your fork
7. Submit a pull request

### PR Description

Include:
- What changed and why
- Any new dependencies or requirements
- Testing performed
- Screenshots (if applicable)

- Help others learn and grow
- Share your experiences and use cases
- Provide feedback thoughtfully
