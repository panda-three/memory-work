# MEMORY.md

长期记忆存储。包含跨周保留的洞见（动态记忆）和情境→行动模式（程序性记忆）。

---

## 写入协议

### 谁来写

- **AI** 在以下场景提议写入：执行模式中发现高惊奇信号（后台标记）、复盘模式中批量扫描提取
- **用户** 可随时手动添加或修改条目

### 写入流程

1. AI 发现值得记录的信号
2. 用自然语言向用户描述发现
3. 用户确认后，AI 写入对应区块
4. 在 MEMORY_LOG.md 记录本次写入操作

### 强度标记

| 符号 | 含义 | 证据要求 |
|------|------|----------|
| ★ | 初始观察 | 单次行为或明确陈述 |
| ★★ | 重复验证 | 2-3 次一致行为 |
| ★★★ | 稳定模式 | 多次跨情境验证，可考虑毕业到 USER.md |

### 衰减规则

- 未被引用的条目每 4 周降一级强度
- ★ 级条目降至无星后，下次复盘时决定保留或清除
- 被主动引用的条目重置衰减计时器
- 标注 `graduated` 的条目不参与衰减

## 动态记忆（Layer 2）

跨周保留的洞见、偏好和决策。这些是关于用户「怎么想」和「在意什么」的记忆。

### 条目格式

```
### [简短标题]
- **强度**：★★
- **发现日期**：YYYY-MM-DD
- **证据**：[具体行为或对话记录的简述]
- **related**：[关联的程序性记忆条目，如有]
- **备注**：[补充说明，如适用场景、例外情况]
```

> `related` 字段用于建立动态记忆与程序性记忆之间的交叉引用。一个洞见可能关联多个行动模式，反之亦然。

- **备注**：收件箱暂存的两条认知（"不想做任务"+"追求即时满足忽略结果"）是此模式的情绪信号。

## 程序性记忆（Layer 3）

情境→行动的模式匹配。当特定情境出现时，AI 按照已确认的模式行动。

### 条目格式

```
### [模式名称]
- **强度**：★★
- **触发情境**：[什么条件下触发]
- **行动模式**：[AI 应该怎么做]
- **发现日期**：YYYY-MM-DD
- **related**：[关联的动态记忆条目，如有]
- **证据**：[发现这个模式的具体场景]
```

### 当前条目

### Panda 的决策节奏：大量输入 → 突然收敛
- **强度**：★★
- **发现日期**：2026-02-25
- **证据**：W09 前半周大量输入（李一舟课程、小七姐方法论、夙愿动态、Dong 哥视频、多条收件箱灵感），周三突然收敛——确定教师提效方向、砍任务、重构四象限、锁定公众号定位。W09 后半段再次出现：周四起锁定 OpenClaw，从广撒网切换为连续 4 天深潜单一工具。
- **related**：→ 动态「任务焦虑的根源与解法」、程序性「深潜模式」
- **备注**：两次验证，升级为 ★★。节奏特征：积累期不急着收敛，但一旦锁定就切换很快。

### 任务清单膨胀时的应对
- **强度**：★★
- **触发情境**：任务清单条目超过 5-6 条未完成，用户表达焦虑或拖延信号
- **行动模式**：1) 用当前里程碑过滤，砍掉不服务当前阶段的任务；2) 学习型任务加明确完成标准；3) 探索型任务冻结而非删除，标注"当前阶段不做"
- **发现日期**：2026-02-25
- **related**：→ 动态「任务焦虑的根源与解法」
- **证据**：用户主动提出"任务越来越杂，看到就想拖延"，经讨论后确定里程碑过滤器为解法

### 新任务入池前的追问过滤
- **强度**：★
- **触发情境**：用户提出新任务要求加入本周任务清单时
- **行动模式**：不直接写入，先追问过滤——比如"这件事不做会怎样？""它帮你通过 M1 吗？""这周必须做还是以后也行？"。问到用户自己确认"确实要做"才写入任务清单，否则不记录。
- **发现日期**：2026-02-26
- **related**：→ 动态「任务焦虑的根源与解法」、程序性「任务清单膨胀时的应对」
- **证据**：用户反思发现很多任务"当时没想清楚就记了，仔细想想压根不需要做"。主动要求 AI 在入池前充当过滤器，用追问帮他筛掉伪需求。

### 深潜模式：锁定目标后连续多日高密度研究
- **强度**：★
- **触发情境**：用户确认某个工具/方向值得深入后
- **行动模式**：识别到深潜信号时，配合节奏——不打断、不插入其他任务建议，帮助维持专注
- **发现日期**：2026-03-01
- **related**：→ 程序性「决策节奏：大量输入→突然收敛」
- **证据**：W09 周四起锁定 OpenClaw，从部署→配置→知识体系搭建，连续 4 天高密度投入单一工具。与前半周广撒网形成鲜明对比。

### 重复收件箱信号 = 未解决的真实痛点
- **强度**：★
- **触发情境**：收件箱处理时发现同一主题重复出现 2 次以上
- **行动模式**：标记为「反复信号」，在复盘时提醒用户关注，建议转化为具体可执行的任务而非继续悬挂
- **发现日期**：2026-03-01
- **related**：—
- **证据**：W09 收件箱中「看视频太长」出现 2 次、「区分电脑/手机任务」出现 2 次，均未被彻底解决，持续冒出。

### 「找最佳实践」变成拖延的挡箭牌
- **强度**：★★
- **触发情境**：面对新任务时，反复搜索别人的做法、最佳实践，迟迟不动手
- **行动模式**：用 AI 快速拿到「60 分起点」，直接动手，在做的过程中迭代。不等找到 90 分方案再启动
- **发现日期**：2026-03-01
- **related**：→ 动态「任务焦虑的根源与解法」[graduated]、程序性「决策节奏：大量输入→突然收敛」、程序性「复杂任务拖延→行动的破局路径」
- **证据**：(1) 用户自述——遇事总想先找别人有没有实践过，找的过程很麻烦导致拖延。(2) 看 OpenClaw 最佳实践时，别人都在炫技，看到大量自己不知道的高级功能 → 产生焦虑。但那些功能跟自己的实际需求无关。焦虑本质：看的东西跟自己的实际不吻合，信息越多越焦虑。
- **备注**：两次验证（自述 + OpenClaw 焦虑实例），升级为 ★★。核心判断标准：「这个最佳实践跟我当前要做的事有关系吗？」没关系就跳过，别看。

### 复杂任务拖延→行动的破局路径
- **强度**：★
- **触发情境**：用户面对复杂任务表现出拖延、恐惧、想放弃的信号
- **行动模式**：按以下顺序帮助破局——
  1. 砍范围：把任务砍到只剩核心产出（「写公众号+一键发布+沉淀Skill」→「写一篇文章」）
  2. 降工具门槛：先用熟悉的工具跑通，再过渡到新工具（Claude Code 先行，OpenClaw 后续）
  3. 给「烂开始」许可：第一次做出来不好用是正常的，先完成再完美
  4. 清场：暂停所有非核心任务，给最高优先级腾心理带宽
- **发现日期**：2026-03-01
- **related**：→ 程序性「找最佳实践变成拖延的挡箭牌」、程序性「任务清单膨胀时的应对」
- **证据**：W09 周日复盘对话。用户从「这个任务好难，想放弃」到最终确定明天动手写公众号文章。转变过程：拆任务→降工具门槛→砍掉其他任务→聚焦唯一重点。完整心路：拖延→恐惧→识别焦虑来源（看别人炫技产生信息焦虑）→拆小→确定行动。

## 使用说明

### 这个文件怎么用

- **AI 读取**：启动时按需检索，不全量加载。根据当前对话情境匹配相关条目。
- **AI 写入**：发现新模式时提议，经用户确认后写入。
- **用户修改**：随时可以直接编辑、删除或调整条目。
- **周复盘**：memory-review 技能会系统性地扫描和更新此文件。

### 与其他文件的关系

- `USER.md`：MEMORY.md 中 ★★★ 级稳定条目的毕业目的地
- `_本周.md`：仅与本周相关的记忆留在周文件中，不进入 MEMORY.md
- `MEMORY_LOG.md`：记录对 MEMORY.md 的每次读写操作
