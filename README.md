# Memory Work v2（中文版）

一套基于 Claude + Obsidian 的 **Personal Agent** 工作方法，帮助你和 AI 建立持续的协作记忆。

> 这是中文完整版本。English version → [memory-work-v2](../memory-work-v2/)

---

## 这是什么

Memory Work 是一个人机协作框架。它让 AI 不只是「一次性对话工具」，而是一个能记住你的偏好、理解你的工作节奏、主动帮你整理和推进的**工作搭档**。

核心理念：
- **四层记忆**：从不变的身份到每周的工作记忆，分层管理
- **周节奏**：以一周为单位的启动→推进→归档循环
- **惊奇度驱动**：AI 只记录真正有价值的发现，不记流水账
- **用户主权**：所有记忆写入需要你的确认，AI 不能偷偷改你的档案

## 快速开始

### 前置条件

- [Claude Desktop](https://claude.ai) + Cowork 模式
- [Obsidian](https://obsidian.md)（推荐，非必须）
- 一个你愿意持续使用的工作文件夹

### 安装步骤

1. **克隆仓库**
   ```bash
   git clone https://github.com/YOUR_USERNAME/memory-work.git
   cd memory-work
   ```

2. **选择版本**
   - 中文版：使用 `memory-work-v2-cn/` 目录
   - English：使用 `memory-work-v2/` 目录

3. **将文件夹设为 Cowork 工作目录**
   在 Claude Desktop 中选择此文件夹作为工作空间

4. **首次启动**
   在对话中发送 `启动` 或 `开始工作`

5. **初始化**
   系统检测到首次使用后，会引导你：
   - 填写 `USER.md`（你的基本信息）
   - 创建第一个周文件
   - 开始你的第一个工作周期

> 详细指南见 [文档/快速上手.md](文档/快速上手.md)

## 系统架构

```
memory-work-v2-cn/
├── CLAUDE.md            ← 系统入口（自动注入）
├── SOUL.md              ← AI 人格定义
├── USER.md              ← 用户画像（你来填）
├── MEMORY.md            ← 长期记忆
│
├── 00 专注区/            ← 周工作台
│   ├── _本周.md          ← 当前周文件
│   ├── MEMORY_LOG.md     ← 记忆日志
│   ├── ITERATION_LOG.md  ← 迭代日志
│   └── _归档/            ← 历史周文件
│
├── 01 素材/              ← 个人档案与方法论
├── 02 工具/              ← 提示词模板与工具
├── 06 技能库/            ← AI 技能定义
└── 文档/                 ← 使用文档
```

### 记忆四层

| 层级 | 内容 | 变化频率 |
|------|------|----------|
| Layer 0 | 身份（SOUL.md / USER.md） | 极少 |
| Layer 1 | 工作记忆（_本周.md） | 每周 |
| Layer 2 | 动态记忆（MEMORY.md） | 跨周保留 |
| Layer 3 | 程序性记忆（MEMORY.md） | 情境触发 |

---

## 工作流程

### 周启动（周一）
口述想法 → AI 拆解任务 → 确认计划

### 周中推进（周二至周四）
做事 → AI 追踪进展 → 遇到阻塞记录

### 周末归档（周五至周日）
回顾产出 → 记忆复盘 → 归档 → 准备下周

---

## 自定义

这套系统设计为**可定制的框架**，而非固定流程：

- **SOUL.md**：调整 AI 的沟通风格和协作边界
- **USER.md**：填入你的实际信息和偏好
- **CLAUDE.md**：修改启动序列和同步规则
- **区域代理**：为新的工作区域创建专属规则
