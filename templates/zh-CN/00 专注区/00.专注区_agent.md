# 00.专注区_agent

> 本区域的核心职责：管理注意力焦点，以周为单位运转。

---

## 区域结构

```
00 专注区/
├── 00.专注区_agent.md     ← 本文件
├── _本周.md               ← 当前周工作文件（永远是这个名字）
├── _模板_本周.md          ← 周文件模板
├── _工作流说明.md         ← 操作流程详情
├── _别名对照表.md         ← 口语称呼映射
├── MEMORY_LOG.md          ← 记忆系统运行日志
├── ITERATION_LOG.md       ← Agent 架构迭代日志
├── _归档/                 ← 历史周记录
├── {{year}}-W{{week}}_日程.ics  ← 本周日历文件
└── [本周临时产出物]
```

---

## 调度规则

### 进入本区域时

1. 优先读取 `_本周.md` 获取当前工作上下文
2. 如需了解工作流程，读取 `_工作流说明.md`
3. 如遇到不明确的称呼，查阅 `_别名对照表.md`

### 周一（周期开始）

- 检查 `_本周.md` 是否已更新
- 如未更新，提醒用户进行周启动
- 根据口述内容拆解任务、搜索全库材料

### 周末（周期结束）

- 提醒用户进行归档
- 确认本周产出物，列出待归档清单
- 确认归档位置，移动文档
- **★ 执行 memory-review**：向用户展示本周记忆操作清单，通过访谈校准记忆策略
- 协助将 `_本周.md` 重命名并移入 `_归档/`
- 创建新的周文件（继承未完成任务 + 记忆系统的迭代动作）

## 日历生成功能

### 触发时机

当用户的 ToDo 讨论完成后，主动引导用户创建日程：

```
任务列表整理好了。我注意到有几个任务带有时间安排：

1. 周三 10:00-12:00 客户会议
2. 周五 14:00 截止 - 提交方案

要我帮你生成一个日历文件，可以直接导入日历软件吗？
```

### 生成流程

1. **确认日程细节**：
   - 具体日期和时间
   - 时长（如未提供，默认 1 小时）
   - 事件标题
   - 备注说明（可选）

2. **生成 .ics 文件**：
   - 文件命名：`{{year}}-W{{week}}_日程.ics`
   - 存放位置：`00 专注区/`
   - 时区：`Asia/Shanghai`

3. **引导用户导入**：
   ```
   日历文件已生成：{{year}}-W{{week}}_日程.ics

   导入方式：
   - macOS 日历：双击文件，或拖入日历 app
   - Google Calendar：设置 → 导入与导出 → 导入
   - Outlook：文件 → 打开和导出 → 导入/导出

   需要我调整任何日程吗？
   ```

### .ics 文件格式

```ics
BEGIN:VCALENDAR
VERSION:2.0
PRODID:-//Memory Work//Claude Cowork//CN
CALSCALE:GREGORIAN
METHOD:PUBLISH
X-WR-CALNAME:{{year}}-W{{week}} 周日程
BEGIN:VEVENT
UID:{{unique-id}}@memory.work
DTSTAMP:{{created-timestamp}}
DTSTART;TZID=Asia/Shanghai:{{start-datetime}}
DTEND;TZID=Asia/Shanghai:{{end-datetime}}
SUMMARY:{{event-title}}
DESCRIPTION:{{event-description}}
END:VEVENT
END:VCALENDAR
```

## 本区域文件说明

| 文件 | 用途 | 操作权限 |
|------|------|----------|
| `_本周.md` | 当前周工作记录（Layer 1 工作记忆） | 可读写 |
| `MEMORY_LOG.md` | 记忆系统运行日志（周复盘结果） | 可追加 |
| `ITERATION_LOG.md` | Agent 架构迭代日志（版本变更记录） | 可追加 |
| `_工作流说明.md` | 工作流程文档 | 只读（修改需确认） |
| `_模板_本周.md` | 周文件模板 | 只读 |
| `_别名对照表.md` | 别名映射 | 可追加 |
| `_归档/` | 历史周记录 | 只读 |
| 临时文档 | 本周产出 | 可读写 |

---

## 与全局规则的关系

- 本代理规则**优先于** AGENTS.md 的通用规则
- 本区域的 `_本周.md` 是全局记忆系统的 Layer 1（工作记忆）
- 本区域的 `MEMORY_LOG.md` 记录记忆系统自身的运行状态
