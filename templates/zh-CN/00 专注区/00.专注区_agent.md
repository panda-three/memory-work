# 00.专注区_agent

> 本区域的核心职责：管理注意力焦点，以周为单位运转。

---

## 区域结构

```
00 专注区/
├── 00.专注区_agent.md     ← 本文件
├── _本周.md               ← 当前周工作文件（永远是这个名字）
├── _模板_本周.md          ← 周文件模板
├── _工作流说明.md         ← 操作流程详情
├── _别名对照表.md         ← 口语称呼映射
├── MEMORY_LOG.md          ← 记忆系统运行日志
├── ITERATION_LOG.md       ← Agent 架构迭代日志
├── _归档/                 ← 历史周记录
└── [本周临时产出物]
```

---

## 调度规则

### 进入本区域时

1. 优先读取 `_本周.md` 获取当前工作上下文
2. 如需了解工作流程，读取 `_工作流说明.md`
3. 如遇到不明确的称呼，查阅 `_别名对照表.md`

### 周一（周期开始）

- 检查 `_本周.md` 是否已更新
- 如未更新，提醒用户进行周启动
- 根据口述内容拆解任务、搜索全库材料

### 周末（周期结束）

- 提醒用户进行归档
- 确认本周产出物，列出待归档清单
- 确认归档位置，移动文档
- **★ 执行 memory-review**：向用户展示本周记忆操作清单，通过访谈校准记忆策略
- 协助将 `_本周.md` 重命名并移入 `_归档/`
- 创建新的周文件（继承未完成任务 + 记忆系统的迭代动作）

---

## 本区域文件说明

| 文件 | 用途 | 操作权限 |
|------|------|----------|
| `_本周.md` | 当前周工作记录（Layer 1 工作记忆） | 可读写 |
| `MEMORY_LOG.md` | 记忆系统运行日志（周复盘结果） | 可追加 |
| `ITERATION_LOG.md` | Agent 架构迭代日志（版本变更记录） | 可追加 |
| `_工作流说明.md` | 工作流程文档 | 只读（修改需确认） |
| `_模板_本周.md` | 周文件模板 | 只读 |
| `_别名对照表.md` | 别名映射 | 可追加 |
| `_归档/` | 历史周记录 | 只读 |
| 临时文档 | 本周产出 | 可读写 |

---

## 与全局规则的关系

- 本代理规则**优先于** AGENTS.md 的通用规则
- 本区域的 `_本周.md` 是全局记忆系统的 Layer 1（工作记忆）
- 本区域的 `MEMORY_LOG.md` 记录记忆系统自身的运行状态
