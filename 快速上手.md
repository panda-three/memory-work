# 快速上手

15 分钟内让 Memory Work 跑起来。

---

## 第一步：准备环境（2 分钟）

### 必须

- **Claude Desktop** 或支持 Cowork 模式的 Claude 客户端
- 一个工作文件夹（就是这个仓库目录）

### 推荐

- **Obsidian**：用于浏览和编辑 Markdown 文件
- **Git**：用于版本控制和备份

---

## 第二步：获取文件（3 分钟）

```bash
# 克隆仓库
git clone https://github.com/YOUR_USERNAME/memory-work.git

# 进入中文版目录
cd memory-work/memory-work-v2-cn
```

或者直接下载 ZIP 并解压。

---

## 第三步：连接 Claude（2 分钟）

1. 打开 Claude Desktop
2. 进入 Cowork 模式
3. 选择 `memory-work-v2-cn/` 文件夹作为工作目录
4. Claude 会自动读取 `CLAUDE.md`

---

## 第四步：首次启动（5 分钟）

在对话框中输入：

```
启动
```

系统会检测到这是首次使用，并引导你完成初始化：

### 4.1 填写用户画像

AI 会引导你填写 `USER.md` 的核心字段：
- 姓名和职业
- 主要工作领域
- 沟通偏好
- 常用工具

> 不需要一次填完。核心身份先填，其他后续补充。

### 4.2 创建首个周文件

AI 会问你：「这周想做什么？」

你可以：
- 直接说出本周的想法和计划
- 粘贴会议笔记或待办清单
- 描述你面对的问题或挑战

AI 将你的输入转化为结构化的 `_本周.md`。

### 4.3 确认周计划

检查 AI 生成的任务清单：
- 修改不准确的地方
- 补充遗漏的内容
- 确认优先级

---

## 第五步：开始工作（3 分钟）

初始化完成后，你的工作流变成：

### 每日

- 做事
- 完成任务后在 `_本周.md` 勾选
- 遇到新想法或阻塞，随时告诉 AI

### 每周

- **周一**：口述想法 → AI 拆解任务
- **周中**：推进工作 → AI 追踪
- **周末**：复盘 → 归档 → 准备下周

---

## 常见问题

### 必须用 Obsidian 吗？

不必须。任何 Markdown 编辑器都可以。Obsidian 的优势是双向链接和文件浏览。

### 可以修改文件结构吗？

可以。这是框架不是规矩。添加新的区域文件夹、调整命名、修改工作流都可以。记得同步更新对应的区域代理文件。

### AI 会自动修改我的文件吗？

- `_本周.md`：是的，AI 会追踪进展并更新
- `USER.md` / `MEMORY.md`：只有经过你的确认才会修改
- `SOUL.md`：需要你的明确同意

### 数据存在哪里？

全部在你的本地文件夹中。没有云端同步，没有第三方服务。你拥有所有数据。

### 怎么备份？

推荐用 Git：
```bash
git add .
git commit -m "weekly backup"
git push
```
